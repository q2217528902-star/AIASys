# AIASys 前端风格与页面骨架

状态: 主线专题
更新时间: 2026-04-04
适用版本: v0.3.9 当前主线

---

## 1. 文档目的

本文档用于收敛 AIASys 当前前端的视觉风格定调、信息呈现方式和基础页面骨架。

它回答的是：

- AIASys 前端整体应该长成什么气质
- 哪些视觉原则必须稳定，哪些表现可以灵活变化
- 主要页面应该采用什么骨架
- 任务工作区、设置中心、任务配置、市场页面如何在结构上保持同一产品语言

本文档不替代具体组件实现，也不直接替代 `frontend-pattern` 的工程规范。它负责回答 AIASys 自己的系统级设计基调。

对应的静态结构示意可参考：

- `.agents/skill-development/aiasys-system-design/examples/task-workspace-shell-v1.html`

---

## 2. 视觉风格定调

### 2.1 总体气质

AIASys 当前前端应保持以下气质：

- 冷静
- 清楚
- 工具化
- 可长期使用
- 面向工作而不是面向宣传

它更接近“长期工作的控制台”，而不是“营销感很强的 AI 展示页”。

### 2.2 关键词

当前视觉关键词应收敛为：

- 中性底色
- 稳定边界
- 轻层次卡片
- 信息优先
- 证据优先
- 少量语义色
- 高密度但可折叠

### 2.3 不应采用的方向

当前不应把 AIASys 做成：

- 大面积装饰性渐变首页
- 强烈品牌秀场风格
- 霓虹感过强的实验 UI
- 依赖大量彩色模块区分功能的控制台
- 每个页面都像独立产品的拼装式视觉

---

## 3. 基础视觉语言

### 3.1 颜色策略

颜色只服务结构和语义，不服务表演。

当前推荐原则：

- 页面背景使用中性浅底
- 卡片、面板、弹窗用轻层次区分
- 边框清晰但不过重
- 语义色只用于状态、提醒、类别和可操作重点

推荐理解：

- 基础结构色：`background / card / border / muted / secondary`
- 状态色：`success / warning / destructive / info`
- 强调色：只保留一组稳定主强调色，不为每个模块单独发明品牌色

### 3.2 字体与排版

排版优先级应是：

1. 可读
2. 稳定
3. 层级清楚
4. 适合高密度长时间阅读

当前建议：

- 标题层级清楚，但不追求夸张大标题
- 正文优先保证密度和扫描效率
- 数值、状态、路径、ID 等技术信息默认用更稳定的对齐方式展示
- 不依赖花哨字体营造风格

### 3.3 间距与密度

AIASys 是工作系统，不是留白导向的展示站。

因此应采用：

- 中高密度信息布局
- 明确分区
- 面板内紧凑但不拥挤
- 列表和弹窗支持折叠、分组、摘要

默认不要把工作区页面做成大量留白的海报式布局。

### 3.4 圆角与阴影

圆角和阴影只用于建立层级。

推荐原则：

- 页面基础面板统一圆角等级
- 弹窗层级比卡片略强
- 阴影保持轻量，不做浮夸悬浮感
- 通过边框和底色层级来表达结构，阴影只是辅助

## 4. 信息展示原则

### 4.1 信息优先于装饰

每个页面都应先回答：

- 用户现在在哪
- 当前主工作对象是什么
- 下一步最常用操作是什么
- 哪些信息是证据，哪些只是说明

如果装饰影响了上述四点，就应回退装饰。

### 4.2 高密度页面必须有折叠策略

以下页面默认要支持折叠或摘要：

- 任务配置
- 设置中心
- 执行记录
- 市场条目列表
- 资源与能力入口

原则是：

- 用户先看到分组摘要
- 需要时再展开细节
- 不把全部技术信息一次性砸在主视区

### 4.3 文案克制

前端文案只写：

- 这是什么
- 这里能做什么
- 当前状态是什么
- 从这里会去哪里

不把以下内容直接渲染到页面里：

- 架构解释
- 重构背景
- 内部治理说明
- “当前真正可编辑的是某某层”这类内部口径句子

## 5. 核心页面骨架

### 5.1 任务工作区骨架

任务工作区是 AIASys 的主页面，应采用稳定三段式骨架：

```text
+--------------------------------------------------------------------------------+
| 顶部栏: 当前任务标题 | 模式切换 | 全局状态 | 快捷操作                             |
+-------------+------------------------------------------------+------------------+
| 左侧任务导航 | 中间主工作区                                   | 右侧任务工作区   |
|             |                                                |                  |
| - 新建任务   |  对话流                                         | - 文件与资料     |
| - 搜索       |  --------------------------------------------  | - 任务资源       |
| - 任务列表   |  当前轮结果 / 中间态反馈                        | - 任务产物       |
| - 设置入口   |                                                | - 工作区摘要     |
|             |  输入区                                         |                  |
|             |  [输入框.........................][发送/运行]    |                  |
+-------------+------------------------------------------------+------------------+
```

页面原则：

- 左侧只承接任务导航和最小系统入口
- 中间是唯一主工作面
- 右侧默认只承接任务工作区
- 执行记录和诊断信息通过二级入口打开，不作为右侧并列一级主区
- 设置和市场不直接占据主工作面

### 5.2 任务配置弹窗骨架

任务配置用于回答“这个任务这次怎么跑”，推荐结构：

```text
+----------------------------------------------------------------------+
| 任务配置                                                              |
+----------------------+-----------------------------------------------+
| 左侧页签导航         | 右侧内容区                                    |
|                      |                                               |
| - Agent 配置         | [摘要卡片] 当前工作区状态 / 最近运行态         |
| - 能力配置           | --------------------------------------------- |
| - 运行环境           |  当前页签内容                                 |
|                      |  - 表单 / 列表 / 推荐项                       |
|                      |  - 折叠组 / 摘要卡                            |
|                      |                                               |
|                      | [保存]                     [关闭]             |
+----------------------+-----------------------------------------------+
```

页面原则：

- 先显示工作区摘要，再进入配置细项
- 高密度能力清单默认分组与折叠
- 不把市场整页直接塞进任务配置

### 5.3 设置中心弹窗骨架

设置中心用于承接用户长期默认配置，推荐结构：

```text
+----------------------------------------------------------------------+
| 设置                                                                  |
+----------------------+-----------------------------------------------+
| 左侧 section 导航    | 右侧内容区                                    |
|                      |                                               |
| - Agent 默认配置     | [当前 section 标题]                           |
| - 模型与偏好         | [一句话说明]                                  |
| - 运行环境           | --------------------------------------------- |
| - Skill 市场         |  当前默认配置项 / 已安装条目 / 可进入入口      |
| - MCP 市场           |                                               |
| - 变量与密钥         |                                               |
|                      | [保存默认项]             [关闭]               |
+----------------------+-----------------------------------------------+
```

页面原则：

- 始终保持“长期默认层”语义
- section 数量变多时，左侧导航必须保持可扫描
- 市场在设置里只是入口和管理区，不应覆盖设置本身的主语义

### 5.4 市场页面骨架

Skill 市场和 MCP 市场都属于目录层，骨架应相似：

```text
+--------------------------------------------------------------------------------+
| 市场标题 | 搜索 | 筛选 | 已安装数量 | 可用数量                                  |
+--------------------------------------------------------------------------------+
| 分类侧栏          | 条目列表                                                   |
|                  |                                                           |
| - 全部            | +-----------------------------------------------------+   |
| - 推荐            | | 条目名称        状态标签                              |   |
| - 已安装          | | 一句话说明                                         |   |
| - 其他分类        | | [查看详情] [安装/配置]                             |   |
|                  | +-----------------------------------------------------+   |
|                  | +-----------------------------------------------------+   |
+--------------------------------------------------------------------------------+
```

页面原则：

- 目录优先，不做复杂工作流页
- 每个条目只保留名称、简介、状态和主操作
- 不写“推荐给谁使用”这类弱信息密度内容

## 6. 页面之间的统一关系

### 6.1 三类主容器

当前前端应稳定为三类主容器：

- 页面型容器：任务工作区、Skill 市场、MCP 市场
- 弹窗型容器：设置中心、任务配置
- 面板型容器：右侧任务工作区、执行记录抽屉、摘要区

### 6.2 同一对象在不同容器中的语义必须稳定

例如：

- `TaskConfigDialog` 永远回答任务级覆盖
- `SettingsDialog` 永远回答用户长期默认层
- 市场永远回答系统提供什么

不能因为容器变了，就让对象语义漂移。

---

## 7. 响应式原则

### 7.1 桌面端

桌面端优先保持三段式结构，尤其是任务工作区。

### 7.2 平板与窄屏

平板和窄屏下应采用：

- 左侧导航折叠
- 右侧辅助区改为抽屉或切换面板
- 主工作区优先保留

### 7.3 移动端

移动端优先保证：

- 当前任务对话与输入区可用
- 设置与任务配置改为全屏抽屉式进入
- 右侧辅助内容通过二级入口切换，不与主区并列硬塞

---

## 8. 当前不推荐的前端模式

当前不推荐继续采用的模式包括：

- 左侧放过多系统功能入口
- 中间主区长期驻留解释型内容
- 每个配置页都采用完全不同的视觉语言
- 市场、设置、任务配置三者混写
- 用大段说明文案代替清楚结构
- 为“酷炫”牺牲信息扫描效率

---

## 9. 本专题的当前结论

当前 AIASys 前端应理解为：

1. 一套中性、稳定、工具化的工作系统界面。
2. 主视觉以结构清楚、信息优先、轻层次卡片和少量语义色为基础。
3. 主工作面优先给任务工作区，设置和任务配置更多通过弹窗完成。
4. 市场页保持目录型页面，不与任务页和设置页混写。
5. 高密度信息默认配合分组、折叠和摘要，而不是无限堆叠。
6. 所有主容器都应共享同一产品语言，不再继续出现拼装式风格漂移。

这份专题文档用于约束后续 AIASys 的页面结构、样式定调和主要容器设计。
