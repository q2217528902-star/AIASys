# 任务配置入口与信息架构

状态: 主线专题
更新时间: 2026-04-04
适用版本: v0.3.9 当前主线

---

## 1. 文档目标

本文档用于回答三个实现前必须先收口的问题：

1. 当前任务的任务配置应该从哪里进入
2. 任务配置内部应该按什么信息层次组织
3. 不同生命周期阶段下，哪些内容允许编辑

本文档属于系统设计与前端信息架构文档，不替代后端配置模型。

---

## 2. 核心结论

### 2.1 任务配置属于当前任务上下文

任务级 Agent、能力、默认执行方式、资源挂载，都是“这个任务这次怎么跑”的问题。

因此任务配置主入口必须放在当前任务工作区上下文内，而不是要求用户跳转到全局设置页。

### 2.2 顶部栏是任务配置主入口

推荐主入口：

- 当前任务顶部栏中的“任务配置”按钮

可接受的次入口：

- 当前任务标题区域快捷入口
- 新建任务完成后的“继续配置”

不推荐作为主入口的位置：

- 左侧全局导航
- 全局设置页
- 右侧工作区内部 Tab

### 2.3 任务配置不承接市场目录

任务配置弹窗只负责当前任务覆盖。

市场目录仍然独立存在，例如：

- Skill 市场
- MCP 市场
- 运行环境目录

任务配置里可以展示已启用摘要和跳转动作，但不应把市场整页塞进弹窗。

---

## 3. 推荐入口布局

### 3.1 当前任务顶部栏分工

顶部栏建议稳定承接以下动作：

1. 任务基础信息
   - 标题
   - 模式
   - 当前主状态摘要

2. 快速运行动作
   - 运行
   - 停止
   - 导出

3. 当前任务配置动作
   - 任务配置

4. 次要操作
   - 更多菜单
   - 删除任务
   - 复制链接

其中“任务配置”应是稳定可见入口，不应藏得太深。

### 3.2 左侧栏职责

左侧栏继续承接：

- 新建任务
- 任务切换
- 全局入口

左侧栏不再承担当前任务 Agent 配置主入口职责。

---

## 4. 任务配置弹窗结构

推荐分成 5 个一级分组，按“从任务语义到执行落地”的顺序组织。

### 4.1 基础

回答“这个任务是什么”：

- 任务标题
- 模式
- 标签
- 目标摘要
- 生命周期状态摘要

### 4.2 Agent

回答“这个任务由谁执行，如何执行”：

- 当前任务 Agent profile
- 提示词版本 / 覆盖
- 工具启用摘要
- 子 Agent 策略
- 配置生效时机

这里必须区分：

- 当前保存配置
- 本轮实际执行配置快照

### 4.3 能力

回答“这个任务挂了哪些能力”：

- Skill 挂载摘要
- MCP 启用摘要
- 能力覆盖来源
- 缺失依赖提示

### 4.4 默认执行方式与运行态

回答“这个任务默认怎么跑，以及当前分支实际上跑成什么样”：

- 工作区默认执行方式
- 当前本地执行链路
- 当前分支运行态摘要
- 环境刷新要求
- runtime 状态摘要

当前主线下不建议在这里直接堆一个“运行环境市场”给用户选，而应先把三层事实讲清：

- 工作区默认执行方式
- 当前分支临时运行覆盖
- 执行轮次运行证据

### 4.5 资源

回答“这个任务能访问什么材料”：

- 文件集
- 数据库连接
- 知识库
- 外部资料挂载

---

## 5. 生命周期下的可编辑性

### 5.1 draft

`draft` 阶段允许最大范围编辑：

- 基础可编辑
- Agent 可编辑
- 能力可编辑
- 环境可编辑
- 资源可编辑

### 5.2 active

`active` 阶段仍允许保存配置，但必须把“保存”与“热更新”分开：

- Agent 允许保存，但默认下一次执行生效
- 能力允许保存，但默认下一次执行生效
- 环境允许切换，但 busy 时应提示等待或重建
- 资源允许调整，但是否立即生效取决于 runtime 刷新策略

### 5.3 completed

`completed` 表示上一轮执行已结束，不表示任务关闭。

因此 `completed` 阶段仍允许继续编辑任务配置。

---

## 6. 前端展示要求

### 6.1 摘要优先

任务配置弹窗默认应先展示摘要，不默认展开全部内容。

每个分组标题建议自带摘要，例如：

- Agent：profile、提示词版本、工具数
- 能力：Skill 数、MCP 数、异常数
- 环境：当前环境、runtime 状态
- 资源：文件数、数据库数、知识库数

### 6.2 文案只说用户需要知道的事

允许的文案：

- 已保存，下次执行生效
- 当前运行中，稍后可切换
- 当前环境需要重建

不应把内部实现解释直接渲染成用户提示。

### 6.3 页面与弹窗职责分离

全局设置页面负责：

- 用户默认值
- 市场目录
- 系统级说明

当前任务弹窗负责：

- 当前任务这次怎么用

---

## 7. 实施顺序建议

1. 在当前任务顶部栏恢复或新增“任务配置”入口
2. 把任务配置弹窗至少拆成“基础 / Agent / 能力 / 环境 / 资源”
3. 把任务级 Agent 配置从全局设置语义中剥离
4. 再决定全局 Agent 配置页只保留为默认值中心还是进一步合并
