### 修复 (Bug Fixes)

#### v0.4.22 发布热修复

- **桌面端 CI 构建失败（release blocker）**
  - `apps/web/src/components/error/BackendCrashOverlay.tsx`：`setDesktopCrashed(info ?? true)` 类型不匹配，改为 `setDesktopCrashed(info ?? { exhausted: true })`。
  - `apps/web/src/types/api.ts`：`SubagentContentEvent` / `SubagentToolCallEvent` / `SubagentToolResultEvent` / `SubagentStepEvent` 补齐 `subagent_name` / `subagent_type` 可选字段，与后端 `wrap_subagent_event` 输出一致。
- **版本与文档同步**
  - 更新 `apps/desktop/package-lock.json`、`apps/web/package-lock.json` 中的版本号为 `0.4.22`。
  - `README.md`、`docs/README.md`、`docs/getting-started.md`、`docs/guides/getting-started/QUICKSTART.md` 同步版本徽章与 Activity Bar 描述。
  - 新增 `images/readme/workspace-layout.png` 工作区三栏布局截图。

#### 子 Agent 与能力确认

- **子 Agent 能力确认死锁（CRITICAL）**
  - 子 Agent 产生 `capability_confirmation` 事件后阻塞在自己的 `CapabilityConfirmationManager` 上，但审批 API 查找的是 Host session 的 manager，导致子 Agent 永久死锁。
  - `sessions_approvals.py`：`_get_runtime_session` 支持 `agent_id` 参数，从 `SubAgentRegistry` 查找子 Agent session 的 `ConfirmationManager`。
  - `capability_confirmation.py`：改用 `threading.Lock` 保护 `_pending` / `_session_auto_approved` 共享状态。
  - 前端 `capabilityConfirmation.ts`：`resolveCapabilityConfirmation` 加可选 `agentId` 参数。
  - 前端 `ChatArea/index.tsx` / `useWorkspacePageController.ts`：审批/拒绝时透传 `agentId`。

- **子 Agent wire 文件 O(n) 读取性能退化**
  - `subagent_events.py`：`_sync_read_wire_records` 从 `seek(0)` + skip 行循环改为 `seek(last_size)`，O(n) → O(1)。

#### 后端安全与鉴权

- **目录端点未鉴权**
  - `/channels/platforms` 与 `/database-connectors/capabilities` 添加 `Depends(require_auth())`。

  - 外部 Skill 市场 `sources` / `items` / `detail` 端点补登录鉴权。

  - `sessions_exports.py`：`import_session_conversation` 加 100MB 大小限制。

- **LLM 服务商 base_url 校验**
  - 新增 `app/utils/llm_url_validator.py`。
  - `llm_config.py`、`models/llm_provider.py` 对 `base_url` 做字段级校验，拒绝非法协议与内网敏感地址。

- **Workspace import 目录遍历**
  - `workspaces_core.py`：`preview_import_folder` 限制 `source_path` 必须位于用户主目录或系统临时目录下。
