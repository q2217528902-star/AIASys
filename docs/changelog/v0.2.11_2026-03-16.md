### 修复会话根目录与执行环境 `/workspace` 挂载漂移

#### 问题描述
- 现象：右侧工作区已经能看到用户文件，但 Docker / Local 执行环境里读不到同一批文件；历史会话里的 `.preference.md` 和旧上传文件还可能停留在 `workspace/` 子目录。
- 原因：文件 API、会话目录初始化、偏好管理和 `IPythonBox` 使用的宿主机工作区根目录不一致。

#### 改动内容
- 后端把会话根目录收敛为当前真实工作区，停止继续把新文件写入 `session/workspace/`。
- `IPythonBox` 挂载会话根目录到容器内 `/workspace`，并在启动前迁移 legacy `workspace/` 子目录中的历史文件。
- `PreferencesManager` 把任务偏好物理路径统一到会话根目录 `.preference.md`，同时兼容旧会话自动迁移。
- 文件列表与会话统计现在都以会话根目录优先，legacy `workspace/` 仅作为回退兼容源。

### 修复前端切换会话时右侧工作区侧边栏不同步

#### 问题描述

### 修复本地沙盒启动与官方镜像初始化校验链路

#### 问题描述
- 现象：Local 模式下内核启动失败；Docker 模式下即便目标官方镜像存在，也可能因为镜像内容漂移而无法执行。
- 原因：`LocalIPythonBox` 仍按旧版 `jupyter_client` 调用方式启动内核；官方环境镜像只做存在性判断，没有做有效性校验和重建闭环。

#### 改动内容
- `LocalIPythonBox` 改为兼容当前 `jupyter_client` 启动方式，并只消费当前 execute 请求对应的 IOPub 消息。
- 官方环境镜像管理器改为以 `environments.json` 为唯一事实来源，按“校验现存镜像 -> 尝试加载 tar -> 按官方 Dockerfile 重建 -> 再校验”的顺序执行。
- 后端启动改为按 `sandbox.default_mode` 分支：默认 `docker` 的部署后台触发官方镜像初始化，不再阻塞 `/health`；默认 `local` 的部署明确跳过 Docker 镜像初始化。
- 部署初始化和更新脚本都补入 `scripts/init_runtime_env_images.py`，保证发布后即可初始化可用的官方镜像。
- `data-analysis.Dockerfile` 修复依赖声明错误，并把当前官方镜像标签统一到 `aiasys/python-data-analysis:1.0.0`。

### 统一沙盒模式配置模型，并让前端只展示真实可选项

#### 问题描述
- 现象：部署实际只支持 Local 或只支持 Docker 时，前端仍可能展示无效入口或伪选择；双模式部署下，新建任务弹窗也可能因为本地缓存和前端兜底值漂移，默认落到错误的沙盒视图。
- 原因：后端仍兼容旧的 `sandbox.mode + allow_local` 语义，前端多个入口分别自行猜测可用模式，没有统一以 `/api/system/capabilities` 返回值为准。

#### 改动内容
- 后端把沙盒配置收敛为 `sandbox.default_mode + sandbox.enabled_modes`，并保留对旧字段的向后兼容解析。
- `/api/system/capabilities` 现在直接返回真实可用模式集合和默认模式；运行环境切换接口会拒绝当前部署未启用的沙盒模式。
- `DesignSidebar`、`InputArea`、运行环境管理面板和新建任务弹窗统一接入 `useSystemCapabilities`，只在 Docker 真正可用时展示运行环境管理入口。
- 新建任务弹窗按部署能力区分三种展示：`local-only` 只提示本地执行，不再要求选择运行环境；`docker-only` 直接选择运行环境，不再额外伪造“沙盒模式选择”；`local + docker` 同时展示两种沙盒卡片，并在切到 Docker 后再显示运行环境列表。
- 新建任务默认沙盒模式现在会优先跟随后端 `default_sandbox_mode`；只有用户存在显式偏好时，前端才回写并沿用本地 `localStorage` 记录。
- `infra/deploy/common.sh` 现在会在远端部署包中显式渲染 `sandbox.default_mode + sandbox.enabled_modes`，避免云端继续继承本地双模式配置。
- 新增浏览器回归用例，覆盖“双模式部署默认显示本地视图，同时可切到 Docker 环境列表”的交互链路，并同步更新已有生命周期用例以兼容新的本地模式弹窗结构。
