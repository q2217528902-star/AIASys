# Changelog v0.4.1 - 2026-06-14

### 修复 (Bug Fixes)

- **修复会话中所有 Assistant 回复都显示 "Turn 1" 的问题**：
  - **流式显示原因**：每次用户发送新消息都会触发一次新的 `/api/agent/stream` 执行入口，执行入口会为事件投影创建全新的 `event_state`，其中 `turn_n` 从 0 开始计数，导致每一轮回复的 Turn 标记都重置为 1。
  - **流式显示解决**：在 `AiasysRuntimeSession` 上暴露 `session_turn_count` 属性（从持久化历史恢复并在每次 ReAct turn 时递增），并在 `execute` 与 `execute_stream` 两个执行入口初始化 `event_state` 时将其设为初始值，使 Turn 编号跨执行连续递增。
  - **历史加载原因**：Assistant 消息写入 session history snapshot 时没有带上 `turn_n`，导致前端历史恢复时退化为 `legacyTurnIndex++`；同时旧数据合并逻辑会把 `turn_n` 同为 `undefined` 的相邻 Assistant 消息合并成一条，进一步让所有历史回复只显示一个 Turn 标记。
  - **历史加载解决**：
    - `_persist_message_to_session_history` 新增 `turn_n` 参数，三个 Assistant 消息落盘点统一写入 `session.session_turn_count`。
    - 前端 `mergeAssistantMessages` 只在两条 Assistant 消息都有明确 `turn_n` 且相等时才合并，避免旧数据被错误合并。
  - **影响文件**：
    - `apps/backend/app/services/agent/runtime_backends/aiasys/session.py`
    - `apps/backend/app/services/agent/mixins/execution.py`
    - `apps/web/src/pages/WorkspacePage/hooks/sessionManagementHistory.ts`
