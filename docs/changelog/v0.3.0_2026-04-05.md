# Changelog v0.3.0 - 2026-04-05

## li-xiu-qi

### 工作区优先的分析主界面与本地运行时成为主链

#### 问题描述
- 现象：分析页长期混杂旧运行时入口、旧壳层和分散的任务配置语义，用户难以把“工作区 / 会话 / 运行态”看成一条清晰主链。
- 原因：前后端仍保留大量旧 Docker 时代和旧入口结构的兼容分支，导致产品骨架和实际主路径不一致。

#### 改动内容
- 后端补齐工作区优先的本地运行时服务，把工作区、知识、数据库、偏好等能力收敛到当前主线接口。
- 前端刷新分析页主壳层，围绕工作区优先结构重排侧边栏、输入区和首页入口。
- 分析页默认围绕当前工作区展开，不再把旧运行时管理概念暴露成主线入口。

### 退役 Docker 主线路径并收口单机默认用户模式

#### 问题描述
- 现象：系统已经以本地执行为主，但前后端仍残留旧 Docker 运行态和登录模式分叉，用户界面会看到与当前主线不一致的入口或判断。
- 原因：历史兼容路径没有一起清掉，认证与运行时主线仍存在旧语义残留。

#### 改动内容
- 后端移除 Docker 运行时主线路径，运行时能力与会话执行统一围绕当前本地主链收口。
- 单机默认用户模式成为清晰主路径，前端路由守卫、用户资料页和首页口径同步更新。
- 旧兼容认证接口和历史登录态对当前单机主链的干扰被移除。

### 修复执行记录实时渲染与 Skill 挂载兼容性

- 修复 Skill 挂载与执行流接口的兼容问题，避免当前主链执行时漏掉新挂载能力。
