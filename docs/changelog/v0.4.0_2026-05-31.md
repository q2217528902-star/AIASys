# Changelog v0.4.0 - 2026-05-31

## ke

### 新增功能 (Features)

- **桌面版 Windows NSIS 安装包**：
  - 支持选择安装目录（`oneClick: false`）
  - 安装前自动检测并关闭运行中的 AIASys Desktop
  - 卸载时询问是否删除用户数据（`%APPDATA%/AIASys Desktop`）
  - 不需要管理员权限（`asInvoker`，避免拖放文件失效）
  - 自定义 NSIS 脚本 `build/installer.nsh`
  - GBK 编码的中文许可协议 `build/license.txt`

- **系统托盘**：
  - 关闭窗口隐藏到系统托盘（不是退出）
  - 托盘右键菜单：显示窗口、打开日志目录、打开数据目录、退出
  - 点击托盘图标切换窗口显示/隐藏

- **Windows 子进程管理**：
  - `windowsHide: true` + `stdio: "pipe"` 避免弹出 cmd 黑窗
  - 崩溃检测：`waitForUrl` 轮询中检查 `child.exitCode`，崩溃时立即报错不 90 秒干等
  - 日志持久化：stdout/stderr 同时输出到控制台和 `{userData}/backend-runtime/logs/{backend|frontend}-spawn.log`

- **Windows 端口占用检测**：
  - `netstat -ano` 获取 PID + `tasklist` 获取进程名
  - 端口被占用时自动回退到下一个可用端口

- **Windows PTY 终端支持**：
  - 后端集成 `pywinpty>=2.0.0`，Windows 上可用 WebSocket 终端
  - 优先查找 PowerShell，回退到 cmd.exe

- **Windows 危险命令拦截**：
  - ShellTool 新增 Windows 特有危险命令检测（`del C:\`、`format`、`diskpart`、`shutdown` 等）

- **执行资源包安装 UI**：
  - UV 环境卡片支持展开/折叠
  - 显示已安装包列表（最多 24 个）
  - 输入框输入包名，支持多个包用空格分隔

### 优化与改进 (Improvements)

- **构建流程**：`prepare-runtime.cjs` 自动清理 `__pycache__` 和 `.pyc` 文件，减小打包体积并避免缓存不一致
- **启动错误处理**：bootstrap 失败时显示错误对话框，包含日志路径，并自动打开日志目录

## 增量更新（2026-06-03）

### 重构 (Refactor)

- **删除旧 DataAnalysisPage**：`apps/web/src/pages/DataAnalysisPage/` 全部清理，代码迁移至 `WorkspacePage`
- **路由统一**：`/analysis` → `/workspace`，保留旧路径兼容重定向
- **知识库去 ORM 化**：元数据与向量均存储在自包含的 `{kb_id}.db`，移除 SQLAlchemy ORM 依赖

### 新增功能 (Features)

- **模板自定义排序**：工作区模板列表支持用户拖拽排序，结果持久化到 UI 设置
- **工作区列表无限滚动**：设计侧边栏历史区域支持分页加载，避免全量加载会话元数据
- **数据库连接器工作区隔离**：连接器支持 `workspace_id` + `scope` 字段，按工作区可见性过滤
- **工作区导出附带对话历史**：ZIP 包包含过滤后的对话记录
- **Agent 工具增强**：
  - `ReadFile` 新增非文本文件拦截（Excel/PDF/Office/图片等），给出替代操作建议
  - `QueryDataTable` 替代已删除的 `ReadDataTableRecords`
  - `ReadMediaFile` 提示词增强，强调优先使用此工具分析媒体内容
  - `BatchCanvasOperations` 描述修正参数格式
- **表格数据预览内置 skill**：`tabular-data-preview-skill`
- **桌面壳增强**：托盘菜单新增打开用户配置/数据目录，Windows pyvenv 路径修复

### 修复 (Fix)

- **会话历史**：合并空白 assistant 消息，避免 ReAct 轮次产生无内容分隔线；新增 `limit` 分页读取
- **Agent 流状态**：流结束后保留 completed/error 状态，支持显式清理
- **Windows 编码**：`monitor_tool` 使用 `smart_decode` 替代硬编码 utf-8

### 文档更新

- 用户文档中 `/analysis` 路由引用统一更新为 `/workspace`
