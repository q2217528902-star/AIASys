# Changelog v0.1.3 - 2026-01-31

##  li-xiu-qi

###  前端架构重构 - 组合模式升级

#### ChatArea - Compound Components 模式
- **重构目标**: 消除 boolean prop 泛滥，提升组件可维护性
- **实现方式**:
  - 拆分 `ChatArea` 为 11 个独立子组件
  - 引入 `ChatAreaContext` (state/actions/meta 三层接口)
  - `ChatAreaProvider` 管理消息状态
  - 子组件通过 Context 获取 `isUser` 状态，消除多处条件判断

- **新增组件**:
  - `MessageAvatar` - 头像显示
  - `MessageBody` - 消息体容器
  - `MessageContent` - 内容容器
  - `MessageLayout` - 布局容器
  - `MessageTimestamp` - 时间戳
  - `UserMessageContent` - 用户消息内容
  - `AiMessageContent` - AI 消息内容
  - `MessageItem` - 消息项组装器

#### AiMessageContent - Explicit Variants + Context
- **重构目标**: 拆分 500+ 行单一组件，提升可测试性
- **实现方式**:
  - 拆分为 10 个独立子组件
  - 每个区域独立：思考区、工具区、回答区
  - 通过 Context 共享 `token`、流式状态等

- **新增组件**:
  - `LoadingPlaceholder` - 加载占位符
  - `StoppedIndicator` - 终止状态提示
  - `ToolBlock` - 工具调用/输出块（可折叠）
  - `FinalAnswerBlock` - 最终回答块（支持导出 Word）
  - `WorkerIndicators` - Worker 活动指示器
  - `ThoughtSection` - 思考内容区域
  - `ToolsSection` - 工具调用区域
  - `AnswerSection` - 最终回答区域

#### MultiTaskSidebar - Context Interface 模式
- **重构目标**: 将 38 个 props 减少到 Context 管理
- **实现方式**:
  - 定义 `SidebarContext` (SidebarState/SidebarActions/SidebarMeta)
  - `SidebarProvider` 隔离状态管理实现细节
  - UI 组件通过 `useSidebarContext()` 获取状态和动作

- **新增组件**:
  - `SidebarContainer` - 容器组件（支持拖拽调整宽度）
  - `TabNavigation` - Tab 导航（执行流/工作区）
  - `TaskStatusBar` - 任务状态条
  - `TaskTimeline` - 事件时间线
  - `FooterStatus` - 底部状态栏
  - `LoadingState` - 加载状态

#### ToolSelectionSheet - Composition Pattern
- **重构目标**: 提供更灵活的组装方式
- **实现方式**:
  - 支持基础用法（传入 props 自动渲染）
  - 支持高级用法（children 自定义布局）
  - 通过 Context 共享选项列表、选中状态等
