# 版本更新记录 v0.0.4

**更新日期**: 2026-01-12

##  li-xiu-qi

###  新增功能 (Features)
- ** 后端对话历史持久化 (Backend History Persistence)**:
  - 引入 `SessionManager` 模块，采用文件系统存储会话元数据与聊天记录。
  - 实现了会话的自动创建与标题自动生成（基于首条消息）。
  - **工作空间映射**: 实现了会话 ID 与 `uploads/{session_id}/` 目录的自动关联，支持统一管理会话及其附件。

- ** History API**:
  - 新增 `/api/history` 路由模块，提供标准的 RESTful 接口供前端调用。
  - 接口支持：会话列表（含附件统计）、会话详情（完整消息）、标题修改、级联删除（同时删除历史与附件）。

- ** Host 模式集成**:
  - `Host` 智能中控模式现已集成 SessionManager。
  - 用户消息与 Host 的流式响应（包括思考过程、任务提交状态）将自动实时保存，确保会话上下文不丢失。

- ** 前端历史会话管理 (Frontend History Management)**:
  - **侧边栏集成**: 左侧侧边栏现可动态展示历史会话列表（支持高亮当前会话）。
  - **无缝切换**: 点击历史会话可即时还原聊天上下文与附件列表，支持删除操作。
  - **状态同步**: 优化了 `useAgentFileUpload` 钩子，支持跨会话切换时的文件列表自动刷新。

- ** 全模式历史记录支持 (All-Mode History Support)**:
  - **覆盖范围**: 除 Host 模式外，现已全面支持 `Workflow` (单一工作流) 和 `Workforce` (多智能体) 的历史记录持久化。
  - **实现策略**: 采用 **请求级汇总 (Request-Level Aggregation)** 策略，在 API 层非侵入式地拦截并保存 "First User Input" 和 "Final Assistant Output"。
  - **架构解耦**: 核心 Agent 逻辑零修改，完全通过 `streaming.py` 和 Router 层的 Middleware 实现。

- ** 历史会话文件导入 (File Import from History)**:
  - 新增从历史会话导入文件的功能（`FileImportModal` 组件）。
  - 支持选择源会话、预览文件列表、批量导入到当前会话。

- ** Workforce 能力清单 (Workforce Capabilities)**:
  - Host 系统提示词新增 Workforce 能力说明，包括数据分析、代码执行、文件处理等核心能力描述。

- ** 交互模式重构与体验优化 (Interaction Mode Refactoring)**:
  - **概念更名**: 将 `OwlMode` 统一更名为 `InteractionMode` (交互模式)，Hook 同步重命名为 `useInteractionModeStream.ts`。
  - **职责解耦**: 明确 Host 模式作为主交互入口，其内容同步至主聊天区域；Workflow 和 Workforce 模式定位为执行模式，内容仅在右侧「执行空间」侧边栏展示。
  - **交互增强**: 为 Workflow/Workforce 模式增加了专属的系统提示消息，在任务提交后自动打开并聚焦右侧执行空间，提升了测试与批量任务执行的体感。

###  优化与改进 (Improvements)
- **Host 交互简化**:
  - 移除 `draft_plan` 工具，Host 现在通过自然语言对话确认用户意图，而非工具调用。
  - 更新 `HOST_SYSTEM_PROMPT`，强调对话式确认流程。

- **前端模块化重构**:
  - 将 `CodeExecutorTestPage.tsx` 拆分为模块化目录结构：
    - `index.tsx` - 主页面逻辑
    - `components/` - ChatArea, InputArea, WelcomeScreen, FileImportModal
    - `hooks/` - useChatState, useSessionManagement, useFileImport
    - `types.ts` - 类型定义
  - 删除旧的单文件 `CodeExecutorTestPage.tsx.bak`。

- **UI 优化 (智谱风格)**:
  - 顶部工具栏采用透明背景，无分割线，视觉更简洁。
  - 模式选择器使用 `appearance-none` + 自定义箭头图标，显得更高级。
  - 新增"查看执行详情"悬浮按钮，方便打开右侧边栏。

- **执行空间显示优化**:
  - 过滤 Host 任务不显示在 `MultiTaskSidebar` 中，只显示真正的 Workforce 执行任务。

###  修复 (Bug Fixes)
- **Host 文件可见性修复**:
  - 修复 `Host._get_or_create_agent` 中 memory 恢复后 system prompt 丢失的问题。
  - 确保导入的历史会话文件能被 Host 正确识别。

- **Host 模式执行空间修复**:
  - 添加 `listenToTask` 调用逻辑，修复 Host 模式下 Workforce 任务无法触发右侧边栏的问题。

###  文档完善 (Documentation)

  - 新增 `Agentbackend/app/session_manager/docs/design.md`: 详细阐述会话管理器的架构设计与存储方案。
  - 新增 `Agentbackend/app/api/docs/History_API_设计说明.md`: 详尽的 API 接口定义与集成时序图。
  - 新增 `Agentbackend/app/api/docs/Workflow_History_集成设计.md`: 针对流式任务的历史集成设计规范。
  - 新增 `frontend/docs/frontend_交互模式设计说明.md`: 阐述 InteractionMode 的设计哲学与交互逻辑。

###  测试 (Tests)
