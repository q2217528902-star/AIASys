# 版本更新记录 v0.0.8
**更新日期**: 2026-01-27

##  li-xiu-qi

###  新增功能 (Features)
- **文件上传进度条**:
  - 在文件上传组件中新增了可视化的进度条，直观展示上传进度。
  - 优化了交互体验，上传过程中自动禁用"发送"按钮，防止重复提交。
  - 底层将 `useAgentFileUpload` 从 `fetch` 切换为 `XMLHttpRequest`，实现了精确的上传进度监听。
- **AI 回复导出 Word 文档**:
  - 新增 `frontend/src/utils/exportDocx.ts` 工具模块，支持将 AI 的最终回答导出为 `.docx` 格式的 Word 文档。
  - 基于 `docx` 库实现纯前端 DOCX 生成，支持完整的 Markdown 格式转换：标题、列表（有序/无序）、代码块、表格、引用块、链接、粗体/斜体/删除线等。
  - **图片嵌入支持**: 自动识别回复中的图片，通过 Vite 代理获取图片数据并嵌入到 DOCX 文件中，解决了跨域（CORS）问题。
  - 在 `AiMessageContent.tsx` 的 `FinalAnswerBlock` 组件中添加"导出 Word"按钮，点击即可下载带图片的完整文档。
  - **Excel 文件预览支持**:
  - 新增 `XlsxPreview` 组件，支持 `.xlsx`、`.xls` 等 Excel 文件的在线预览。
  - 实现了多工作表（Sheet）切换浏览功能。

###  优化与改进 (Improvements)
- **前端 API 配置统一化 (Unified Configuration)**:
  - 新增 `frontend/src/config/api.ts` 集中管理 API 基础 URL，支持通过 `.env` 中的 `VITE_API_BASE_URL` 灵活配置（开发环境默认走 Proxy，生产环境可指定 IP）。
  - 全面重构前端代码（包括 `SessionReportsModal`、`useSkills`、`backendAuthService` 等），移除了所有硬编码的 `localhost:8001`，确保统一使用配置。
  - 提取 `stripApiBaseUrl` 到 `urlUtils.ts`，确保导出文档和图片加载时自动剥离绝对路径，强制请求通过 Vite 代理，彻底解决跨域与端口不一致问题。
- **前端 API 代理机制修复**:
  - 修复了登录、注册等认证请求直接指向硬编码后端地址 (`http://localhost:8001`) 而绕过 Vite 代理的问题。
  - 更新 `BackendAuthService.ts`，将默认 `API_BASE_URL` 设为空字符串，强制使用相对路径以触发 Vite 开发服务器的代理逻辑。
  - 从 `frontend/.env` 中移除了 `VITE_BACKEND_URL` 配置，确保开发环境下请求通过 `http://localhost:7999/api` 转发，解决了跨域及开发环境配置不一致的问题。
- **图片获取代理优化**:
  - 优化 `fetchImageAsBuffer` 函数，将绝对路径 URL（如 `http://localhost:8001/api/files/...`）自动转换为相对路径（`/api/files/...`），确保请求走 Vite 代理，避免 CORS 跨域问题。
- **文件预览性能优化 (CSV/XLSX)**:
  - 针对大文件和宽表文件预览，实现了**智能截断渲染**逻辑。
  - **行/列双向截断**: 仅渲染前 50 行/列 和 后 50 行/列，中间部分使用省略号代替，即使是超大表格也能秒级渲染且不卡顿浏览器。
- **后端中文文件名下载修复**:
  - 修复了后端 `download_routes.py` 中处理中文文件名时的编码错误（`HTTP 500`）。
  - 遵循 **RFC 5987** 规范，对 `Content-Disposition` 中的文件名进行 URL 编码，同时兼容 `filename` 和 `filename*` 属性，确保各浏览器都能正确识别中文文件名。

###  依赖更新 (Dependencies)
- 新增 `docx` 库：用于在浏览器端生成 Word 文档
- 新增 `file-saver` 库：用于触发文件下载
- 新增 `@types/file-saver`：TypeScript 类型定义
- 新增 `xlsx` 库：用于前端解析 Excel 文件结构与数据
