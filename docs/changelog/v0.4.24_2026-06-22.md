# Changelog v0.4.24 - 2026-06-22

## li-xiu-qi

### 修复 (Bug Fixes)

#### 桌面版创建项目转圈 / 卡死（v0.4.24 核心修复）

- **数据库 schema 兼容性**
  - `apps/backend/app/core/database.py`：启动时自动补齐 `Base` 中所有已存在表的缺失列与索引，解决旧 DB 因缺少新增列导致创建项目时后端 500、前端持续转圈的问题。

- **LLM 本地地址校验过严**
  - `apps/backend/app/utils/llm_url_validator.py`：桌面应用默认使用本地/私有地址（`127.0.0.1`、`localhost` 等），原校验器将本地地址误判为内网敏感地址并拒绝。
  - 仅保留对云 metadata 端点（`169.254.169.254`、GCP/Oracle metadata）的拦截，允许本地模型服务正常配置。
  - 新增 `tests/test_llm_url_validator.py` 覆盖本地地址放行与敏感地址拦截场景。

- **Vendor 二进制打包路径查找**
  - `apps/backend/app/core/vendor_binaries.py`：同时支持源码布局与桌面打包布局，避免打包后因 `backend_root` 定位错误找不到 `fnm`/`uv`/`sqlite-vec` 二进制。

- **桌面服务管理器健壮性**
  - `apps/desktop/src/service-manager.cjs`：后端启动日志解析容错增强，启动失败时更准确地向渲染进程报错。

#### Windows 安装包构建失败

- `apps/desktop/build/installer.nsh`：
  - 修复多行字符串未使用 `$
` 转义导致的 `unterminated string` 错误。
  - 删除未使用的 `longPathDone` 标签，消除 NSIS `warning 6012`（在 electron-builder 中被视为错误）。
- `apps/desktop/scripts/download-fnm-binary.cjs`、`download-uv-binary.cjs`、`download-sqlite-vec-binary.cjs`：校验已有二进制时优先判断可执行性，减少 CI 重新下载失败。

#### 其他同步修复

- **后端 API 与稳定性**
  - `database_connectors.py`、`data_tables.py`、`files_core.py`、`knowledge.py`、`sessions_exports.py`、`workspaces.py` 等：补齐异常链、鉴权、路径桥接、并发控制。
  - `graphrag/core/sqlite_graph_store.py`、`knowledge/sqlite_kb_service.py`：修复并发写入、连接池与事务边界问题。
  - `memory/pipeline.py`：调整记忆整理写入边界，避免大容量时阻塞。

- **前端体验**
  - `WorkspaceSidebar` 各面板、CapabilityPanel、RoleDetailDialog、ExportWorkspaceDialog 等：修复类型与交互细节。
  - `useCodeExecutor` 流式事件处理：增强错误兜底。
