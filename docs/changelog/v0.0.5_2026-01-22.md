# 更新日志 - v0.0.5 (2026-01-22)

##  统一会话管理与多 Agent 历史重构

本次更新彻底重构了后端的会话持久化层，解决了 `smolagents` 引擎与主框架存储不一致的问题，并实现了历史记录的“现场级”完美还原。

###  架构改进 (Architecture)
- **SessionManager 统一化**：
  - 废弃了冗余的 `SmolSessionManager`，所有模式（Host/Worker/Smol）统一接入 `Agentbackend/app/session_manager`。
  - 数据目录统一规范为 `conversation_history/{session_id}/`，采用结构化存储（Metadata + ChatHistory）。
- **多 Agent 身份绑定机制**：
  - 在 `ExecutionStep` 中引入 `message_id` 和 `agent_name` 强绑定，解决了历史加载时 Host 与 Worker 内容混淆的问题。
  - 实现了 **增量保存 (Incremental Save)** 机制，通过 `offset` 追踪，避免多轮对话时历史数据无限重复叠加。

###  记忆与持久化 (Persistence)
- **现场还原 (State Restoration)**：
  - 重构了 `load_smol_steps` 逻辑，支持从所有历史消息中回溯汇总完整的 Memory Steps。
  - 实现了“活的上下文”：点击历史会话后，Agent 引擎会自动加载之前的思维链，支持输入“下一个问题”继续之前的执行任务。
- **结构化消息解耦**：
  - 主聊天界面 (`StructuredMessage.content`) 现在只存储干净的最终结论。
  - 所有的思考过程、代码块、执行日志被隔离在 `steps` 字段中，通过侧边栏独立渲染。

###  前端体验与工程化 (Frontend & Engineering)
- **代码组织模块化重构**：
  - **组件分类 (Functional Grouping)**：按功能将 `src/components` 划分为 `auth`、`chat`、`execution`、`file`、`layout` 和 `tool` 六大模块。
  - **页面结构规范化**：将所有页面（Pages）统一重构为文件夹模式，实现了私有逻辑的物理隔离。
- **一致性审计与协议对齐**：
  - **请求头规范化**：修正了 `useInteractionModeStream` 与 `useMultiTaskEventStream` 中的 SSE 连接逻辑，强制要求在 Header 中携带 `X-Session-Id`，确保前后端会话隔离的一致性。
  - **执行空间还原**：强化了历史会话的“现场级”复刻逻辑，通过原子事件拆解精准还原 AI 的思考与执行路径。
- **文档体系深度补齐**：
  - 新增 `frontend_API对接与协议规范.md`：确立了与后端通信的标准协议与异常处理机制。
  - 新增 `frontend_流式渲染与事件流逻辑.md`：详细定义了 Host 与 Worker 的事件分流逻辑与渲染生命周期。
  - 重构 `frontend_交互模式设计说明.md`：确立了“三栏式”布局设计哲学与任务空间交互标准。
  - 更新 `frontend_文件组织指南.md`：同步了重构后的功能模块化目录规范。


###  资源访问与工作目录 (Image & Workspace)
- **空间对齐与 404 修复**：
  - 统一了 `JupyterKernelInterpreter` 的工作目录与资源存储路径至 `uploads/{session_id}/`。
  - 解决了生成的图片由于存储在 `workspace` 影子目录而导致前端路由无法访问的 404 错误。
  - `SessionManager.get_workspace_dir` 现在默认对接 `uploads` 目录，确保资源产出即刻可访问。

###  流式输出分流 (Streaming Routing)
- **主页内容降噪**：
  - 引入了 `WorkerOutputEvent` 协议事件，实现了 Host（主持人）与 Worker（执行者）的消息分流。
  - **Chat 频道**：仅展示 `AgentOutputEvent` (Host 消息)，保持主对话界面的简洁与对话感。
  - **Workspace 频道**：Worker 的思考、代码块和执行日志全部重定向至 `worker_output` 事件，仅在侧边栏执行空间展示。
  - **消除重复**：彻底解决了 Worker 与 Host 同时汇报结论导致的“重画一张图、重说一遍话”的问题。

###  语义化流式渲染与分段 UI 协议 (Segmented UI Streaming)

本次更新确立并实现了“分段式语义渲染”与“严格空间隔离”标准，解决了思维链展示杂乱和界面内容重复的问题。

- **Host 角色指令强化 (Backend)**：
  - 重写了 `HOST_INSTRUCTIONS`，确立【思考归 Thought，结论归 Final Answer】原则。
  - 严禁 Host 在最终回复中包含决策推导、自言自语或内心戏，确保用户收到的回复专业且整洁。
- **分段式 UI 协议实现 (Frontend)**：
  - **动态分段渲染**：前端 `AiResponseCard` 现在根据 `content_type` (`thought` vs `final_answer`) 动态拆分消息。
  - **自动折叠机制**：Host 的思考过程（Thought）自动包裹在带图标的灰色折叠块中，而正式回复则作为无干扰的正文展示。
- **严格空间隔离路由 (Space Isolation)**：
  - **侧边栏 Worker 专享**：侧边栏（Workspace）现在强制仅渲染 `agent_role: "worker"` 的日志和代码，彻底移除了 Host 的干扰信息。
  - **历史记录智能分流**：在加载历史会话时，系统会自动将步骤按角色路由至主频道或侧边栏，完美还原执行现场。
- **流式渲染鲁棒性优化**：
  - **去重逻辑**：修复了 `agent_output` (累积器) 与 `agent_output_delta` (增量) 事件可能导致的文字重复显示问题。
  - **清理冗余逻辑**：移除了前端所有基于字符串猜测的启发式识别，完全信任后端语义化标注。
- **可视化调试工具桩升级**：
  - 更新 `test_data_analysis_api.py`，支持彩色语义化输出（灰色思考/绿色结论/黄色日志/蓝色结果），并能实时追踪工作区文件更新，极大提升了协议联调效率。
