# Changelog v0.4.23 - 2026-06-21

## li-xiu-qi

### 修复 (Bug Fixes)

#### 新建工作区转圈/卡死

- **根因**：`session_budget.py` 中 `_metadata_lock` 是 `asyncio.Lock`，但 `_save_budget`、`_save_context_tokens_to_metadata`、`_check_session_budget`、`_is_session_budget_blocked` 都是同步方法，内部使用 `with self._metadata_lock:`，真实运行时会抛出 `TypeError` 并导致后端进程崩溃，前端所有请求超时转圈。
- 修复：
  - `apps/backend/app/services/agent/runtime_backends/aiasys/session_budget.py`：上述 4 个方法改为 `async def`，锁操作改为 `async with`。
  - `apps/backend/app/services/agent/runtime_backends/aiasys/session_stream.py`：调用方加 `await`。
  - `apps/backend/app/services/agent/mixins/execution.py`：同步模式保存 budget 时加 `await`。
  - `apps/backend/tests/test_session_budget_context_tokens.py`：单测同步改异步，锁使用真实 `asyncio.Lock`。

### 优化 (Improvements)

#### 工作区创建性能与进度反馈

- **后端异步初始化运行时资源**
  - `apps/backend/app/services/workspace_registry.py`：
    - `create_workspace` 在写元数据、对话后立即返回，不再同步执行 `uv sync` / `fnm install`。
    - 新增 `_initialize_workspace_resources`，在后台线程中完成 Python / Node 环境创建与绑定，并持久化进度到 `workspace.json` 的 `initialization` 字段。
    - 修复 Node 默认环境 ID 判断错误：把 `node-default` 误写成 `workspace-default`，导致 Node 工作区初始化直接失败。
  - `apps/backend/app/models/workspace.py`：新增 `WorkspaceInitializationStatus` 模型。
  - `apps/backend/app/api/routes/workspaces_core.py`：新增 `GET /api/workspaces/{workspace_id}/initialization` 查询接口。
  - `apps/backend/tests/test_workspace_registry_route.py`：更新测试以等待异步初始化完成。

- **前端实时进度条**
  - `apps/web/src/lib/api/workspaces.ts`：新增 `getWorkspaceInitialization` API。
  - `apps/web/src/config/api.ts`：新增 `WORKSPACE_INITIALIZATION` 端点。
  - `apps/web/src/pages/WorkspacePage/hooks/useWorkspaceRuntimeControls.ts`：普通创建工作区后，对需要默认 Python/Node 环境的工作区进入 `waiting_runtime` 阶段，每 800ms 轮询初始化状态并更新进度与 message。
  - `apps/web/src/components/NewWorkspaceDialog/NewWorkspaceProgressBanner.tsx` / `index.tsx`：展示后端返回的实时步骤文案。
  - `apps/web/src/types/workspace.ts` / `utils/newTaskLifecycleState.ts`：`NewTaskLifecycleState` 支持 `message` 字段。
