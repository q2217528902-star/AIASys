### 修复 Host Final Answer 图片无法显示问题

#### 问题描述
Host（智能中控）返回的最终回答（final_answer）中，通过 Markdown 引用的图片无法显示，显示为占位符。而 Worker 工具输出中的图片可以正常显示。

#### 根本原因
`AiMessageContent` 组件的 Context meta 对象中未传入认证 `token`，导致 `FinalAnswerBlock` 渲染 Markdown 图片时，无法为图片 URL 附加 `access_token` 参数，后端因此拒绝返回图片内容。

#### 修复方案
在 `AiMessageContent` 组件中：
1. 使用 `useState` 和 `useEffect` 从 `authService.getSession()` 异步获取 token
2. 将 token 传入 `AiMessageMeta`，使 `FinalAnswerBlock` 可以正确获取并使用 token

#### 文件变更
- `apps/web/src/components/chat/AiMessageContent/index.tsx`: 添加 token 获取逻辑

### 修复 Host 回复内容显示问题

#### 问题描述
- **现象**: 前端报错 `Objects are not valid as a React child (found: object with keys {type, loc, msg, input, ctx})`
- **原因**: 
  - 后端返回的 `final_answer` 有时是结构化对象（如 `{"task_outcome_detailed": "..."}`），未正确转换为字符串
  - Pydantic 验证错误 (422) 的 `detail` 是数组格式，前端直接渲染导致报错

### 修复登录 422 错误处理

#### 问题描述
- **现象**: 登录请求返回 422 Unprocessable Entity，前端渲染错误对象
- **原因**: Pydantic 验证错误的 `detail` 是数组格式 `[{type, loc, msg, input, ctx}]`，直接渲染导致 React 报错

#### 修复方案
- 统一使用 `extractErrorMessage()` 处理所有后端错误响应
- 支持提取数组中第一条错误消息的 `msg` 字段
