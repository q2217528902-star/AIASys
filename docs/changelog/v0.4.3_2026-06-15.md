# Changelog v0.4.3 - 2026-06-15

## li-xiu-qi

### 新增功能 (Features)

- **执行资源组：工作区可勾选 Python/Node/Docker 组合**
  - 工作区配置支持按资源类型启用/禁用运行环境，避免 Agent 调用未安装的运行时。
  - 影响文件：
    - `apps/backend/app/services/workspace_registry.py`
    - `apps/backend/app/models/workspace.py`
    - `apps/web/src/components/NewWorkspaceDialog/index.tsx`
    - `apps/backend/tests/test_execution_resource_group.py`

- **启动时自动检查并下载缺失 vendor 二进制**
  - 后端启动与桌面打包阶段自动补齐 uv、fnm 等外部依赖，减少首次运行失败。
  - 影响文件：
    - `apps/backend/app/core/vendor_binaries.py`

### 修复 (Bug Fixes)

- **Windows 下 UV 运行时命令改用 cmd /c 并注入 uv PATH**
  - **现象**：Windows 桌面版运行 UV 环境时，`uv run ... sh -lc` 因无 sh 而失败；子进程也可能找不到 `uv`。
  - **原因**：`wrap_shell_command_for_runtime` 统一使用 POSIX `sh -lc`，未针对 Windows 解释器做适配；`PATH` 未显式包含 uv 所在目录。
  - **解决**：
    - Windows 上 UV 包装命令改用 `cmd /c`。
    - `build_runtime_shell_env` 自动将 `uv` 可执行文件所在目录追加到 `PATH`。
  - 影响文件：
    - `apps/backend/app/services/runtime/runtime_execution.py`
    - `apps/backend/tests/test_runtime_execution.py`

- **Windows 任务栏图标使用 nativeImage 并设置 AppUserModelID**
  - 解决打包后任务栏图标显示为默认图标或文字的问题。
  - 影响文件：
    - `apps/desktop/src/main.cjs`

- **桌面版 pyvenv.cfg home 使用相对路径**
  - 避免用户机器上 Python 绝对路径与打包环境不一致导致环境失效。
  - 影响文件：
    - `apps/desktop/scripts/prepare-runtime.cjs`

### 重构 (Refactor)

- **统一 Shell/Monitor 跨平台执行层到 ShellExecutor**
  - 新增 `ShellExecutor`，集中处理解释器探测（bash/Git Bash/PowerShell/cmd）、路径转换、环境合并、超时控制与进程树清理。
  - `ShellTool` 与 `MonitorTool` 全部迁移到 `ShellExecutor`，移除散落在工具中的子进程创建逻辑。
  - POSIX 子进程启用 `start_new_session`，避免 `killpg` 误伤父进程/测试进程。
  - Windows 进程树清理使用 `taskkill /T`，支持超时后强制终止。
  - 影响文件：
    - `apps/backend/app/services/shell_executor.py`
    - `apps/backend/app/agents/tools/shell_tool.py`
    - `apps/backend/app/services/agent/runtime_backends/aiasys/tools/monitor_tool.py`
    - `apps/backend/tests/test_shell_executor.py`
