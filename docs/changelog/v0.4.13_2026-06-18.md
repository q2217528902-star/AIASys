# Changelog v0.4.13 - 2026-06-18

## li-xiu-qi

### 新增功能 (Features)

- **PDF 翻译工作区模板**
  - 新增 `pdf-translate` 专属工作区模板，用户创建工作区时可直接选择"PDF 翻译"，一键获得翻译环境
  - 预装 3 个精简 Skill：pdf-translate-skill、pymupdf4llm-pdf-to-markdown-skill、aiasys-tool-usage-skill
  - 预置 README.md（使用指南）和 AGENTS.md（翻译工作流指引），开箱即用
  - 相比 paper-reading 模板（7 个 Skill + 论文笔记框架），更轻量、更聚焦翻译场景
  - 模板路径：`apps/backend/templates/pdf-translate/template.toml`

### 优化与改进 (Improvements)

- **pdf-translate-skill 翻译脚本优化**
  - 流式输出：从 `capture_output=True` 改为 `Popen` 逐行透传，翻译过程中实时显示 pdf2zh 的下载和翻译进度，不再黑屏等待
  - 页码范围翻译：新增 `--pages` 参数，支持 `1-5`、`1,3,5` 等格式，大文件可先试译前几页确认效果
  - Python 版本兼容：`uvx` 命令固定 `--python 3.12`，避免在 Python 3.14 上 pydantic-core 因 PyO3 不兼容而编译失败
  - 翻译开始时输出 `translating` 状态消息，包含源文件、翻译器、页码范围信息
  - 错误信息增强：输出目录内容列表辅助排查、保留最后 20 行日志用于错误诊断

- **pdf-translate-skill SKILL.md 更新**
  - 补充 `--pages` 参数说明和使用示例
  - 新增"大文件渐进翻译工作流"指引
  - 补充流式输出说明

### 修复 (Bug Fixes)

- **translate.py 路径解析 bug**
  - 现象：`/workspace/foo.pdf` 被解析为 `workspace/foo.pdf`（多了一层目录），导致"文件不存在"错误
  - 根因：`Path(*agent_path.parts[1:])` 对 `/workspace/foo.pdf` 的 parts `('/', 'workspace', 'foo.pdf')` 去掉首元素后得到 `('workspace', 'foo.pdf')`，没有正确去掉 `/workspace/` 前缀
  - 修复：改为字符串前缀匹配 `raw_path.startswith("/workspace/")` 直接截取剩余路径

### 文档完善 (Documentation)

- 更新 `docs/guides/workspace/workspace-templates.md`：内置模板列表从 6 种更新为 7 种，新增 PDF 翻译模板，移除不存在的"代码开发"模板
- 更新 `docs/guides/agent/agent-chat.md`：PDF 翻译段落修正输出描述（从"Markdown 文件"改为"mono/dual PDF"），补充翻译服务说明
- 更新 `docs/guides/capabilities/mcp-skill-market.md`：pdf-translate-skill 描述补充"输出单语版和双语对照版"

### 测试 (Tests)

- 模板加载验证：`list_workspace_templates()` 返回 7 个模板，pdf-translate 出现在列表中
- API 验证：`GET /api/workspace-templates` 和 `GET /api/workspace-templates/pdf-translate` 返回正确数据
- 工作区创建验证：用 pdf-translate 模板创建工作区，3 个 Skill 正确安装到 `.aiasys/skills/`，capabilities.toml 正确生成
- 端到端翻译验证：测试 PDF → google 翻译 → mono.pdf + dual.pdf + manifest.json 全部生成成功
