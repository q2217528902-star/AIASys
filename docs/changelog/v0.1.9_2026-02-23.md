# 版本更新记录 v0.1.9

**更新日期**: 2026-02-23

## li-xiu-qi

### 新增功能 (Features)

- **IPythonBox 执行流历史同步**:
  - 后端新增 `IPythonBox.get_history()` 类方法，从 Docker 容器拉取累积的代码执行记录
  - 新增 `/execution/{user_id}/{session_id}/flow` API 端点，提供 Notebook 风格的执行历史
  - 前端 `syncNotebookHistory()` 方法，在会话切换时同步历史执行流
  - `useSessionManagement.ts` 新增 `extractExecutionEvents()`，从 `context.jsonl` 解析 IPythonBox 调用

- **ImageLightbox 通用图片预览组件**:
  - 支持点击背景/空白处关闭
  - ESC 键快捷关闭
  - 滚轮缩放（0.25x - 5x），以鼠标位置为中心缩放
  - 拖拽平移查看大图
  - 下载按钮支持原图下载
  - 统一聊天区和文件树的图片预览体验

- **文件树图片缩略图**:
  - `FileTreeView` 新增 32x32px 内联缩略图预览
  - 点击缩略图打开 Lightbox 查看大图
  - 图片类型识别（png, jpg, jpeg, gif, webp, svg）

### 优化与改进 (Improvements)

- **执行流侧边栏标签统一**:

- **Docker Sandbox 文档完善**:

- **代码质量**:

  - `AgentService` 补充 `tool_name` 字段到 SSE 事件

- **遗留代码清理（前端）**:

### 修复 (Bug Fixes)

- **执行流过滤修复**:
  - 统一历史加载和实时流的事件格式（统一使用 `tool_start`/`tool_output`）
  - 修复执行流显示非 IPythonBox 工具（Shell、WriteFile 等）的问题
  - 统一数据源：历史回顾时只从 `/flow` 接口加载，不再从 `context.jsonl` 提取
  - 过滤非 IPythonBox 事件：移除 `subagent_content`、`subagent_step_begin`、`status` 等

- **SSO 模式历史加载**:
  - 移除 `useSessionManagement.ts` 中阻塞性的 `X-User-ID`/`Authorization` 头检查
  - Cookie-based SSO 模式下允许正常加载历史会话

- **历史记录 AI 消息显示修复**:
  - 修复 `MessageItem.tsx` 导入错误的 `AiMessageContent` 组件问题
  - ChatArea 内应使用 `./AiMessageContent`（通过 Context 获取数据）
  - 而非 `../AiMessageContent/index`（通过 props 接收数据）
  - 确保历史会话回顾时 AI 消息内容正常显示

- **tool_result 工具名称显示修复**:
  - 修复流输出中 `tool_result` 事件的 `tool_name` 始终显示为 "unknown" 的问题
  - 根本原因：`ToolResult` SDK 对象不包含 `tool_name` 字段
  - 解决方案：在 `execute_stream` 中维护 `tool_call_map`，保存 `tool_call_id -> tool_name` 映射
  - 现在 Shell、WriteFile 等工具结果能正确显示工具名称

- **工具调用详情执行结果修复**:
  - 修复点击工具调用气泡显示"等待执行结果"，但实际结果已返回的问题
  - 根本原因：`ChatSegment` 没有保存 `tool_call_id`，无法匹配 `tool_call` 和 `tool_output`
  - 解决方案：
    - `useStreamEventHandler.ts`: `tool_call` 和 `tool_output` segment 添加 `toolCallId` 字段
    - `ToolsSection.tsx`: 点击时使用 `toolCallId` 而非 `toolName` 作为标识符
    - `DataAnalysisPage`: 悬浮窗显示正确的工具名称

### 文档完善 (Documentation)

  - 保留工具：Task（子 Agent）、SetTodoList（待办）、ReadMediaFile（媒体读取）

- **新增架构文档**:
  - `execution-flow-design.md`: 执行流面板与工具气泡的职责划分
  - `image-preview.md`: ImageLightbox 组件设计文档

- **文档迁移**:
  - `kimi-sdk-guidelines.md` 第6节迁移至 `execution-flow-design.md`

- **执行流设计文档更新**:
