## li-xiu-qi, zzddkk

### 新增代码执行超时全链路配置

#### 问题描述
- 现象：代码执行超时值此前分散在工具参数和前后端局部逻辑里，用户无法稳定地按会话维度配置，也容易出现 Local / Docker 两条链路行为不一致。
- 原因：超时配置没有成为会话级事实源，前端也缺少明确的输入与范围校验。

#### 改动内容
- 后端新增会话级 `code_timeout` 配置与统一校验逻辑，`LocalIPythonBox`、`IPythonBox` 和本地 Python 工具都改为消费同一套超时配置。
- `POST /sessions`、运行环境切换接口和系统能力接口都补齐了超时相关字段，前端新建任务弹窗可以直接编辑并校验超时值。
- Docker 沙盒执行链路改为 deadline / interrupt 驱动，不再让 LLM 直接通过工具参数绕过超时治理。

### 新增 Agent 配置隔离与前端配置页

#### 问题描述
- 现象：用户无法稳定保存自己的 Agent 提示词与工具偏好，只能依赖系统默认配置或局部临时修改。
- 原因：缺少用户级 Agent 配置存储、权限边界和完整的前端编辑入口。

#### 改动内容
- 后端新增用户级 Agent 配置服务与 `/api/agent-config` 路由，支持提示词合并策略、工具开关和管理员默认配置维护。
- 登录态补齐角色字段与管理员权限判断，前端新增 Agent Config 页面、预览面板和重置交互。
- 动态 Agent 配置生成链路改为读取用户隔离配置，不再把所有人都绑到同一套默认口径。
