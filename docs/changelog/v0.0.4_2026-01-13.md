# 版本更新记录 v0.0.4

**更新日期**: 2026-01-13

##  贡献者ID
- li-xiu-qi

###  新增功能 (Features)
- **Skill 动态集成 (Skill Dynamic Integration)**:
  - 实现 `activate_skill` 工具，支持 Agent 在运行时按需加载专业知识 SOP 与 API 文档。
  - 采用两阶段提示词策略（Discovery & Activation），有效控制 Token 消耗，避免上下文膨胀。
- **结构化执行历史持久化 (Structured Execution History Persistence)**:
  - 引入 `ExecutionSummary` 模型，支持将复杂的多 Agent 执行流（步骤、产出物）持久化为 JSON。
  - 升级 `SessionManager` 核心逻辑，支持执行摘要的独立读写与历史回溯加载。
- **统一流式处理管线 (Unified Stream Pipeline)**:
  - 重构 `useInteractionModeStream` 为无状态解析器，统一 Host 与 Owl/Workforce 的异构事件处理。
  - 标准化 Host 模式下的工具调用与思考过程展示，消除了显示混乱。

###  优化与改进 (Improvements)
- **Owl 单体 Agent 模式重构**:
  - 将 "Workflow" 概念全面升级为 "Owl Single Agent" 模式，优化 API 路由与会话元数据标识。
- **代码库深度精简**:
  - 移除了 `AlignmentCard`, `WorkforceStatusCard` 等 10 余个冗余组件及测试页面。
  - 治理过时 Hooks（如 `useSqlQuery`），解耦逻辑并安全迁移全局类型定义。
- **类型系统强化**:
  - 新增 `file.ts` 统一管理 `TableFileRef` 等文件类型，解决跨组件循环依赖。

###  修复 (Bug Fixes)
- **流式累积逻辑修正**: 修复了 Owl 多轮对话时输出内容在前端被错误覆盖或消失的问题。
- **API 路径对齐**: 修正了历史会话管理中的所有路径错误（`/api/sessions` -> `/api/history/sessions`）。
- **稳定性修复**: 修复了 `InputArea` 文件上传组件的崩溃问题，以及 422 接口调用异常。

###  文档完善 (Documentation)
- **自动化提示词沉淀**: 新增 `docs/ai-coding-prompts/frontend_code_cleanup_prompt.md`，规范前端清理流程。

###  测试 (Tests)
- **功能验证**: 新增 `test_mcp_skill_activation.py`，验证了 MCP 技能激活的闭环流程。
