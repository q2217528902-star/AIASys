# 版本更新记录 v0.0.8

**更新日期**: 2026-01-26

##  li-xiu-qi

###  新增功能 (Features)
- **前端资源鉴权增强**:
  - 新增 `frontend/src/utils/urlUtils.ts` 工具，提供 `appendAccessToken` 函数，用于为 API 请求 URL 自动追加 `access_token`。
  - 更新 `AiMessageContent.tsx` 和 `StreamingThoughtBlock.tsx`，在 Markdown 渲染中自定义 `img` 组件，确保图片加载时携带认证 Token。
  - 更新侧边栏渲染器 (`StandardAgentRenderer.tsx`, `CodeAgentRenderer.tsx`) 和执行历史 (`ExecutionHistoryView.tsx`)，确保所有生成的图片和文件链接均经过鉴权处理。
  - 更新 `SessionReportsModal.tsx`，在获取会话列表和文件列表时注入 `Authorization` Header，并在文件下载/预览链接中追加 Token。
  - 更新 `ToolCard.tsx`，确保工具图标 URL 携带认证 Token。

###  优化与改进 (Improvements)
- **鉴权代码清理与统一**:
  - 删除了不再使用的 `frontend/src/lib/auth/SupabaseAuthService.ts`。
  - 明确了前端鉴权架构：`AuthContext` 依赖 `BackendAuthService` 进行认证，复用 `LocalStorageAuthService` 中的类型定义。
  - 统一了文件和图片资源的访问方式，通过 Query 参数传递 Token，解决了浏览器原生请求无法设置 Header 的问题。
- **工作区样式统一 (Design System Alignment)**:
  - 重构 `WorkspaceArtifacts.tsx` 和 `FileTreeView.tsx`，将硬编码颜色 (如 `slate-*`, `amber-*`) 替换为 Shadcn/UI 语义化 Tokens (`bg-background`, `text-muted-foreground`, `bg-primary/10` 等)。
  - 统一使用 `font-mono` 显示文件名、路径、统计数字等技术信息，与 HomePage 的极简科技风格保持一致。
  - 修复 `fileTreeUtils.ts` 中的路径解析问题，正确处理 Windows 反斜杠路径分隔符。
- **工作区浏览体验升级 (Workspace UX Overhaul)**:
  - **Tree View 重构**: 将原有的面包屑导航升级为 VS Code 风格的**递归树形视图**，支持无限层级缩进和文件夹折叠/展开。
  - **双栏预览模式**: 引入左右分栏布局，点击文件时左侧文件树自动收缩，右侧展示文件预览面板。
  - **多格式预览支持**: 
    - **图片**: 支持 PNG/JPG/SVG 等格式预览，内置放大/缩小/重置控制及下载功能。
    - **CSV**: 内置轻量级 CSV 解析器，以表格形式渲染数据预览。
    - **其他文件**: 为 Notebook (`.ipynb`) 等不支持预览的文件提供友好的文件详情页和下载入口。
  - **交互优化**: 移除了 SessionID 的强制检查限制，确保在任何视图状态下点击文件均能响应并显示详情。
- **全平台图片交互统一化 (Unified Image Experience Overhaul)**:
  - **组件收编**: 新增 `MarkdownImage.tsx` 通用组件，取代了散落在各处的原生 `<img>` 和旧版预览逻辑，实现了 Host 消息、执行流（Execution Stream）以及工作区文件预览的交互完全统一。
  - **深层交互能力**: 预览窗口支持 **0.25x - 5.0x** 的超大范围滚轮及按钮缩放、鼠标拖拽平移、一键重置视图以及原图下载。
  - **代码架构精简**: 删除了 `ImagePreviewModal.tsx` 以及侧边栏中冗余的点击监听与冒泡处理逻辑，大幅提升了 UI 响应的准确性。
  - **自适应布局**: `MarkdownImage` 支持通过 `className` 自定义样式，在保持功能一致的前提下，完美适配了对话流（居中限宽）和工作区预览面板（铺满容器）等不同布局需求。
- **后端执行历史与存储优化**:
  - 优化了 `session_manager.py` 的存储逻辑：在保存步骤时自动过滤冗余的 `final_answer` 工具调用，并实现 Host 连续思考块（Thought）的自动合并，使历史记录更精简且易于回顾。
  - 强化了记忆恢复机制：在 `load_smol_steps` 中支持从消息正文自动补全结论步骤，确保多轮对话上下文逻辑完整。
  - 调整了图片资产存储方案：统一将生成的图片保存至 `workspace/outputs/images` 目录下，并同步更新了 Jupyter 内核解释器与流式适配器的路径转换逻辑。
- **前端历史回顾体验优化**:
  - 修复了 `AiMessageContent.tsx` 在历史恢复时内容缺失的问题：现在支持从所有 Host 步骤（包括 `tool_use` 类型的 `model_output` 字段）中提取并合并思考内容。
  - 彻底删除了前端组件中针对旧版纯文本数据的兼容代码，统一了基于 `steps` 和 `segments` 的结构化渲染流程。
- **Worker 环境感知能力**:
  - 在 `JupyterExecutorAdapter` 中新增 `get_environment_summary()` 方法，可返回当前 Kernel 的环境配置摘要（操作系统、已配置的中文字体、负号修复等）。
  - 在 `DataAnalysisTeam` 初始化 Worker 时，将环境摘要动态注入到 Worker 的 `instructions` 中，使模型在生成代码时能感知到环境已配置的内容，避免生成冗余的字体配置代码。
- **前端规范文档化**:
  - 在 `AGENTS.md` 中新增「前端设计系统 (Design System)」章节，明确了 Color Tokens、Typography、Component Style 的使用规范。

###  修复 (Bug Fixes)
