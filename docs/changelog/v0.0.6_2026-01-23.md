# 版本更新记录 v0.0.6 (Pre-release)

更新日期: 2026-01-23

贡献者ID: li-xiu-qi

##  核心架构升级

### 1. 存储目录标准化 (Storage Standardization)
- **路径重构**: 全面废弃扁平的 `uploads/` 目录，启用分层级的 `storage/` 结构。
  - **会话存储**: `storage/sessions/{session_id}/`
    - `workspace/`: Agent 的工作目录（包含 `input/` 上传文件, `output/` 产出文件）。
  - **运行时存储**: `storage/jupyter_runtime/` (Jupyter Kernel 连接文件)。
- **目的**: 为即将到来的文件版本控制（Versioning）和快照（Snapshot）机制提供物理隔离的目录结构。

### 2. 流式协议优化 (Streaming Protocol)
- **Worker 状态增强**:
  - 新增 `WorkerEndEvent`: 明确标识 Worker 任务结束，包含 `duration_ms` (耗时) 和 `status` (success/failed)。
  - `streaming_adapter` 升级:
    - 独立追踪 Worker 的生命周期（Start -> Output -> End）。
    - 修复了 Worker 执行报错时可能导致流中断或状态丢失的问题。
- **执行摘要 (Execution Summary)**:
  - `EventsToSummaryConverter` 适配新的事件结构，支持更精确的代码执行产物（Artifacts）提取。
  - 修复了从 `tool_input` 中提取代码块的逻辑。

##  依赖与环境

- **数据分析栈**: 默认集成 `pandas`, `numpy`, `matplotlib`, `seaborn`, `scipy`, `scikit-learn`, `plotly`。
- **网络工具栈**: 默认集成 `requests`, `beautifulsoup4`, `lxml`。
- **其他**: 引入 `Pillow` 处理图像，`openai` 作为标准 LLM 客户端依赖。

##  文档更新

- **架构文档**: 更新了 `02_architecture`, `04_storage`, `06_deploy` 下关于文件存储路径的描述。
- **设计文档**: 更新了 Jupyter 执行器与图片嵌入的设计说明，对齐新的存储路径。
