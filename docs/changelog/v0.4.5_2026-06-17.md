# Changelog v0.4.5 - 2026-06-17

## li-xiu-qi

### 新增功能 (Features)

- **会话历史持久化重构：history.json 作为唯一消息源**
  - 彻底移除 `context.jsonl` 作为消息持久化通道，`history.json` 成为会话消息的唯一真实来源。
  - 每轮执行后同步完整运行时消息列表（含 `tool_calls` 和 `tool` 消息）到 `history.json`，修复会话重建后工具调用链丢失的关键缺陷。
  - 移除 `mixins/history.py` 中 ~170 行 mmap 尾读逻辑和 `_should_skip_context_message()` 过滤。
  - 影响文件：`services/agent/mixins/execution.py`、`services/agent/mixins/history.py`、`services/agent/runtime_backends/aiasys/session.py`、`services/session/core.py`、`services/session/history.py`、`api/routes/sessions.py`

- **SSE 心跳保活与 Agent 流自动重连**
  - 后端 SSE 流新增 15 秒心跳机制（`asyncio.Queue` + 独立心跳任务），前端 45 秒超时检测后自动重连（最多 2 次，3 秒延迟）。
  - 前端暴露 `isReconnecting` 状态供 UI 展示重连进度。
  - 影响文件：`api/routes/agent.py`、`hooks/useAgentStream.ts`

- **MCP 服务自动重连**
  - `MCPClient.call_tool()` 捕获连接异常后自动执行 `reconnect()` 并重试，新增 `is_connected()` / `reconnect()` 方法。
  - `/mcp/status` 接口返回运行时 MCP 客户端实际连接状态（`runtime_connected`），不再始终返回 `configured`。
  - `mcp/manager.py` 新增按 `user_id` 的线程锁，防止并发写覆盖。
  - 影响文件：`runtime_backends/aiasys/mcp_client.py`、`api/routes/mcp.py`、`mcp/manager.py`

- **桌面端后端崩溃自动重启**
  - `service-manager.cjs` 重写进程管理：非正常退出时自动重启（最多 3 次，2 秒延迟），通过 `onCrash` / `onRestart` 回调通知渲染层。
  - 新增 `BackendCrashOverlay.tsx` 全屏遮罩组件，后端崩溃/重启期间展示状态。
  - 影响文件：`apps/desktop/src/service-manager.cjs`、`apps/desktop/src/main.cjs`、`apps/desktop/src/preload.cjs`

- **主题系统（亮色/暗色/跟随系统）**
  - 新增 `ThemeContext.tsx` 和 `ThemeToggle.tsx`，支持 `light` / `dark` / `system` 三种模式，localStorage 持久化。
  - 终端组件支持双主题色板切换，新增 WebGL 渲染器（Canvas 降级）。
  - 影响文件：`contexts/ThemeContext.tsx`、`components/ThemeToggle.tsx`、`components/terminal/TerminalPanel.tsx`、`main.tsx`

- **聊天历史前向分页加载**
  - 后端 `/sessions/{id}/history` 支持 `before` 参数实现向前分页，返回 `has_more_before` 和 `oldest_loaded_index`。
  - 前端滚动到顶部时自动加载更多历史消息，保持滚动位置。
  - 影响文件：`api/routes/sessions_execution.py`、`hooks/useSessionManagement.ts`、`components/chat/ChatArea/index.tsx`

- **终端 Tab 持久化与恢复**
  - 终端标签页状态持久化到 `sessionStorage`，刷新后自动恢复并重连 PTY。
  - 关闭的终端可在标签栏「恢复终端」下拉中重新打开。
  - PTY 在关闭标签页时不杀进程，仅移除 UI。
  - 影响文件：`usePaneTree.ts`、`components/WorkspaceTabBar.tsx`、`PaneRenderer.tsx`、`hooks/useTerminal.ts`

- **运行时错误友好分类**
  - `_build_user_facing_runtime_error()` 将错误分为 8 类（context_length_exceeded / auth / forbidden / billing / model_not_found / rate_limit / server / timeout），返回中文友好提示。
  - 影响文件：`services/agent/mixins/execution.py`

- **图片处理增强**
  - 单轮图片上限 10 张，内联图片大小上限 10MB（对齐 Claude API）。
  - 超大图片在 hydrate 时自动缩放（>2048px），PNG 保留透明度，其他转 JPEG quality 85。新增 `Pillow` 依赖。
  - 压缩时图片降级为文本引用 `[历史图片: {path}]`，不再完全丢弃。
  - 缺失图片文件替换为文本占位符，不发送无效 `file://` URI。
  - 影响文件：`services/agent/message_content.py`、`services/agent/compaction.py`

- **文件上传进度显示**
  - `useAgentFileUpload.ts` 改用 `XMLHttpRequest` 支持上传进度，输入框展示进度条。
  - 影响文件：`hooks/useAgentFileUpload.ts`、`pages/WorkspacePage/components/InputArea.tsx`

- **busybox-w32 流式下载**
  - `shell_environment.py` 新增异步生成器 `install_busybox_w32_streamed()`，前端通过 SSE 展示下载进度百分比。
  - 影响文件：`services/shell_environment.py`、`api/routes/system.py`、`components/settings/ShellEnvironmentPanel.tsx`

- **知识库文档自动刷新**
  - 文档处理中状态时每 3 秒自动轮询文档列表。
  - 影响文件：`components/KnowledgeBaseMarket/index.tsx`

- **新增 UI 组件**
  - `EmptyConversationState.tsx`：空会话引导状态（示例问题 + 上传提示）。
  - `SetupChecklist.tsx`：空工作区首页 3 步设置清单（模型配置 / Shell 环境 / 创建工作区）。
  - `PreflightCheck.tsx`：新建工作区对话框中的环境就绪检查。
  - `NetworkStatusOverlay.tsx`：全局网络状态横幅（离线告警 / 恢复确认）。
  - `SectionErrorFallback.tsx`：区块级错误降级组件。

### 优化与改进 (Improvements)

- **WorkspaceRuntimeBinding 简化为 resources-only**
  - `sandbox_mode` 和 `env_id` 不再作为存储字段，改为从 `resources` 派生的 `@property`。
  - 所有消费方更新为只传递 `resources`，`workspace_registry.py` 新增旧格式迁移逻辑。
  - 前端 `types/workspace.ts` 类型同步更新。
  - 影响文件：`models/workspace.py`、`services/workspace_registry.py`、`services/node_runtime.py`、`services/runtime_environment.py`

- **AuthContext 跨标签页同步与 401 全局处理**
  - `httpClient.ts` 全局 401 拦截 → 派发 `aiasys:auth-expired` 事件 → AuthContext 清空状态并跳转。
  - `storage` 事件监听实现跨标签页登出同步。
  - 离线检测：请求前检查 `navigator.onLine`。

- **atomic_write_text 原子文件写入**
  - `path_utils.py` 新增 `atomic_write_text()`（临时文件 + `os.replace()`），用于配置投影、历史快照和会话写入。

- **Task Plan 缓存 LRU 淘汰**
  - `session/task_plan.py` 缓存改用 `OrderedDict` LRU（上限 200 条）+ TTL（300 秒），防止无限增长。

- **SQLite 并发访问**
  - `core/database.py` 新增 `PRAGMA busy_timeout=30000`，防止并发访问时 "database is locked"。

- **Mermaid 渲染性能**
  - `MermaidBlock.tsx` 包裹 `React.memo()`，新增 `useMermaidComponents()` hook 缓存组件配置，避免流式输出时全量重解析。

- **ErrorBoundary 增强**
  - 支持自定义 `fallback` 渲染属性和 `resetError()` 方法。

- **Skill 文档更新**
  - `.team-skills/aiasys-git-workflow/SKILL.md`：新增 WSL git HTTPS fallback 说明和 beta 发布检查清单。
  - `.team-skills/aiasys-tool-dev/SKILL.md`：新增跨平台 shell 工具设计章节。

### 修复 (Bug Fixes)

- **会话重建丢失工具调用上下文（关键）**
  - 现象：会话重建后 LLM 丢失所有工具调用历史，上下文断裂。
  - 原因：`_persist_message_to_session_history()` 只保存最终 assistant 文本，`tool_calls` 和 `tool` 消息全部丢失。
  - 修复：`_sync_session_messages_to_history()` 写入完整运行时消息列表。

- **纯 tool_calls 的 assistant 消息被丢弃**
  - `_load_persisted_messages()` 跳过 `content=None` 的消息，导致纯工具调用轮次丢失。修复为检查 `tool_calls` 字段后再决定跳过。

- **会话删除后刷新"复活"**
  - 前端状态清理在 API 调用之前执行，API 失败时会话会在刷新后重现。修复为 API 成功后再清理前端状态。

- **子 Agent SSE 事件丢失**
  - `isRunning` 依赖导致 SSE handler 在运行状态切换时重建，丢失过渡期事件。修复为提取到 `useCallback` + ref。

- **工作区删除遗留 Docker 容器和 GraphRAG 缓存**
  - 新增 `_cleanup_workspace_containers()` 和 `_cleanup_workspace_graphrag_cache()`，在 `rmtree()` 前清理。

- **文件上传污染工作区根目录**
  - 非 notebook 文件改为保存到 `uploads/` 子目录。

- **Skill 安装竞态条件**
  - 临时目录名改用 `.{name}.install.{uuid}` 避免碰撞。

- **IME 输入法回车误提交**
  - 添加 `!event.nativeEvent.isComposing` 检查，防止 CJK 输入法组合中回车提交。
