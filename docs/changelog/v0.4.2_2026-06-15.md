# Changelog v0.4.2 - 2026-06-15

### 新增功能 (Features)

- **内置 StepFun Search MCP（可安装连接器）**：
  - 新增 `apps/backend/capability_sources/builtin/mcp/stepfun-search/manifest.toml`
  - Server 定义直接写在 manifest 的 `[server_definition]` 段，安装时复制到工作区 `mcp_config.json`
  - 默认不自动启用、不加入 analysis 模式基线，需用户在"能力管理 → 连接器"中手动安装
  - 服务端点：`https://api.stepfun.com/step_plan/v1/mcp/web_search/mcp`
  - 提供 `web_search` 全网搜索与 `web_fetch` 网页内容获取能力
  - 支持通过环境变量 `STEPFUN_API_KEY` 或工作区 Headers 配置鉴权

- **新增 Agent 连接器安装能力**：
  - 新增工具 `SearchAvailableConnectors`：搜索 AIASys 内置源仓库中的可用连接器
  - 新增工具 `InstallConnector`：将指定连接器安装到当前工作区
  - 新增系统内置 Skill `aiasys-connector-installer-skill`：指导 Agent 搜索、选择并安装连接器
  - `InstallConnector` 已加入高风险工具名单，smart 模式下需用户确认

- **Agent Test Case 支持环境变量注入**：
  - `design-draft/agent-tests/run_case.py` 新增 `env_substitutions` 配置
  - case 可在 `case-config.json` 中声明需要从环境变量替换的 key，`run_case.py` 复制 fixtures 后自动完成 `${VAR}` 占位符替换
  - 默认复制的 Skill 增加 `aiasys-connector-installer-skill`

- **MCP Provider 增强**：
  - 安装 MCP 连接器时优先读取 manifest 自身的 `[server_definition]`，找不到再回退到系统默认配置

- **MCP 连接器展示名称修复**：
  - `MCPServerConfig` 新增 `display_name` 字段
  - 能力注册表现在使用 `display_name` 而非 `name` 作为连接器展示名

- **MCP Headers 环境变量解析**：
  - `app/services/llm/mcp_session_service.py` 新增 `${VAR}` 占位符解析
  - 系统默认配置可在 Headers 中使用 `Authorization: Bearer ${STEPFUN_API_KEY}`，运行时自动从 server.env 或进程环境变量取值
  - MCP 连接测试接口同步支持该解析，保证"测试连接"与真实会话行为一致

### 测试 (Tests)

- **新增 Agent 测试用例 TC-MC-004**：
  - 路径：`design-draft/agent-tests/l1-capability-tests/cases/mc-004-stepfun-web-search/`
  - 验证 Agent 在需要实时信息时调用 `web_search` 工具并返回搜索结果
  - 运行前需配置 `STEPFUN_API_KEY`
