# 全局工作区

![全局工作区资源面板](../../../images/readme/panel-global-workspace.png)

## 概念

全局工作区是跨工作区共享的资源空间。每个用户有一个全局工作区，路径为 `workspaces/{user_id}/global_workspace/`。放在全局工作区里的文件在所有普通工作区里都能访问到，不需要复制，不需要手动挂载。

## 与普通工作区的区别

| | 普通工作区 | 全局工作区 |
|------|-----------|-----------|
| 用途 | 单个任务的独立空间 | 跨任务共享资源 |
| 文件 | 任务专属文件 | 公共配置文件、复用脚本、API 文档模板 |
| 知识库 | 任务专属知识库 | 全局知识库（公司文档、产品数据库） |
| 数据库连接 | 任务专属连接 | 全局数据库连接 |
| 图谱 | 任务专属图谱 | 全局图谱（跨项目复用） |
| 访问方式 | 左侧 Activity Bar "当前工作区" | 左侧 Activity Bar "全局工作区"（第二个图标） |

## 访问方式

1. 在 `/workspace` 页面左侧 Activity Bar 点击第二个图标（地球图标）
2. 展开全局工作区面板，可以看到：
   - 全局文件树：跨工作区共享的文件
   - 全局知识库：所有工作区都能使用的知识库
   - 全局数据库连接：公共数据库连接配置
   - 全局图谱：跨项目复用的知识图谱

## 全局资源

### 全局文件

在全局工作区里上传或创建的文件，在任意普通工作区的文件树中都能看到。适合放：
- 公共配置文件（API 规范、代码风格指南）
- 复用脚本（数据处理模板、报告生成模板）
- API 文档模板
- 公司/团队共享的参考资料

### 全局知识库

在全局工作区创建的知识库对所有工作区可用。适合放：
- 公司产品文档库
- 团队技术规范库
- 公共参考资料库

### 全局数据库连接

配置一次，所有工作区都能使用。适合放：
- 公司数据仓库连接
- 公共数据源连接

### 全局图谱

跨项目复用的知识图谱，如公司组织架构图、产品依赖关系图。

## 配置继承

普通工作区继承全局工作区的默认配置，也可以单独覆盖或禁用某项配置。配置优先级为：会话配置 > 当前工作区配置 > 全局工作区配置。

## 文件树中的展示

在普通工作区的文件树中，全局工作区的文件与当前工作区的文件在同一棵树中显示，但通过不同的图标或分组标识区分来源。Agent 文件工具通过 `/global/` 前缀访问全局工作区内容，通过 `/workspace/` 前缀访问当前工作区内容。
