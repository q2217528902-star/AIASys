# 环境变量

环境变量用于管理 API 密钥、连接字符串、代理设置等敏感配置，Agent 在执行代码和 shell 命令时会自动注入，不需要手动 export。

![环境变量注入与验证](../../../images/readme/demo-env-vars-overview.png)

## 两级注入

环境变量分两级管理：

| 层级 | 作用域 | 典型用途 |
|------|--------|----------|
| 全局环境变量 | 所有工作区 | API 密钥、代理设置、LLM provider 配置 |
| 工作区环境变量 | 当前工作区 | 数据库连接串、项目路径、临时 token |

**优先级**：工作区环境变量覆盖全局环境变量。同名变量在工作区层设置后，Agent 在对应工作区中只读到工作区层的值。

## 管理面板

左侧活动栏点击"环境变量"图标打开管理面板。面板提供可视化编辑器，支持：

- 查看和编辑全局环境变量
- 查看和编辑当前工作区环境变量
- 新增、修改、删除变量
- 变量名和值的明文编辑

全局环境变量也可以在 workspace 设置区域中打开全局变量编辑对话框，不需要切换到特定工作区。

## 注入机制

Agent 执行代码或 shell 命令时，系统自动从环境变量存储中读取对应层级的值，注入到执行上下文中。用户和 Agent 都不需要手动 export。注入发生在服务端，变量值不会暴露给前端。

## 安全注意

- 环境变量存储在服务端，不在前端持久化
- 变量值在前端编辑器中以密码字段形式展示，默认隐藏
- 不建议在环境变量中存储超过 10KB 的值，大文件或证书建议通过工作区文件管理
