# 工作区创建与管理

工作区是 AIASys 的一等对象。每个工作区有独立的文件系统、运行环境、会话记录和资源配置。

## 创建工作区

进入 `/workspace` 页面，点击页面中间或左侧边栏的"新建工作区"按钮，弹出创建面板。

创建面板包含以下字段：

| 字段 | 说明 |
|------|------|
| 标题 | 工作区名称，创建后可修改 |
| 描述 | 可选，简要说明工作区用途 |
| 创建方式 | 空白任务 / 从模板创建 / 从文件夹导入（桌面版） |
| 模板 | 从内置模板中选择，决定工作区的初始文件结构和预配置 |
| Python 环境 | 选择工作区绑定的 Python 运行环境 |
| 初始会话标题 | 创建工作区后自动创建的会话名称，默认"新会话" |

### 内置模板

| 模板 | 说明 |
|------|------|
| 空白 | 空工作区，不包含任何预设文件或配置 |
| 数据分析 | 预置数据分析常用目录结构和 Pandas/Matplotlib 依赖 |
| 文献综述 | 预置论文管理目录、文献检索 skill 和知识库配置 |
| 项目开发 | 预置 src/tests/docs 目录结构和 Git 初始化 |

### Python 环境选项

| 选项 | 说明 |
|------|------|
| 无 Python | 不绑定 Python 环境，适用于纯文档类工作区 |
| UV Python | 使用 UV 管理的工作区独立 Python 环境（推荐） |
| 已注册内核 | 从系统已注册的 Jupyter 内核列表中选择 |

### 从文件夹导入

支持选择一个本地文件夹，将其内容复制到新工作区。

1. 在创建面板中选择「从文件夹导入」。
2. 点击「选择文件夹」：
   - 桌面版：调用系统文件对话框。
   - Web 版：通过浏览器选择文件夹（依赖 `webkitdirectory` 支持，推荐 Chrome/Edge）。
3. 系统扫描文件夹后展示文件树，默认排除 `.git/`、`node_modules/`、`.env`、缓存文件等。
4. 在文件树中勾选要导入的文件，点击「创建工作区」。
5. 创建过程中会显示实时进度条（桌面版直接复制，Web 版先上传再复制），完成后自动切换到新工作区。

导入采用复制方式，原文件夹不会被修改。大文件（超过 100MB）默认不选中，需要手动勾选。

### 创建进度

提交创建后，页面顶部会出现进度横幅，依次显示：

1. 准备工作区
2. 扫描并复制文件夹内容（仅文件夹导入）
3. 应用模板文件
4. 初始化 Python 环境（如选择）
5. 创建初始会话

全部完成后横幅自动消失，工作区自动切换为新创建的工作区。

## 切换工作区

左侧边栏顶部显示当前工作区名称。点击名称，弹出工作区列表，列出所有已有工作区。点击目标工作区即可切换。

切换后中间画布和右侧会话侧栏会刷新为对应工作区的内容。

## 工作区设置

点击左侧边栏底部的齿轮图标进入工作区设置。设置面板包含以下分组：

### LLM 模型

- 选择工作区默认使用的 Chat 模型
- 可选模型来自全局模型配置中已添加的服务商
- 会话级配置可以覆盖此设置

### Python/UV 运行时

- 查看和修改工作区绑定的 Python 环境
- 查看已安装的包列表
- 通过 UV 安装或移除 Python 包
- 切换 Python 版本（需 UV 支持）

### Docker 沙盒资源

- 查看和管理工作区关联的 Docker 容器
- 配置容器的 CPU/内存限制
- 查看容器运行状态和日志
- 手动启动/停止/重启容器

### 资源挂载

- 将外部目录挂载到工作区文件系统
- 支持只读和读写两种模式
- 挂载的目录在工作区文件树中显示为独立节点

## 删除工作区

在工作区设置面板底部点击"删除工作区"，输入工作区标题确认后删除。

删除操作不可逆，会移除工作区目录下的所有文件、会话记录和配置。全局工作区不可删除。

## 从模板创建

模板是一键式工作区初始化方案。选择模板创建时，系统自动完成：

- 创建预设的目录结构
- 生成初始配置文件（如 `.aiasys/` 下的配置）
- 安装模板指定的 Skill 和 MCP 服务
- 设置环境变量默认值
- 可选：安装指定的 Python 包

创建完成后可以直接开始工作，不需要手动搭建环境。

模板创建的工作区与手动创建的工作区没有本质区别，后续可以自由修改任何配置。
