# 知识图谱

知识图谱是 AIASys 内置的实体关系管理系统，基于 Property Graph 模型，每张图谱是一个独立的 SQLite 数据库文件（`.graph.db`）。

![知识图谱实体关系探索](../../../images/readme/demo-knowledge-graph-exploration.png)

## 基本概念

知识图谱采用 Property Graph 模型，核心元素：

- **实体**：图谱中的节点，代表一个事物（论文、人物、概念、项目等）。每个实体有唯一 ID 和一组属性（键值对）
- **关系**：实体之间的有向连线，描述两个实体之间的关联（引用、依赖、包含等）。关系也可以带属性
- **社区**：通过 Leiden 算法自动检测的实体聚类，揭示图谱中的结构分组

## 图谱文件

每张知识图谱存储为一个 `.graph.db` 文件，本质是 SQLite 数据库，内部包含实体表、关系表和社区检测结果。图谱文件存放在工作区或全局工作区目录下，在文件树中以资源节点形式展示。

## 图谱工作台

打开 `.graph.db` 文件后进入图谱工作台，核心视图是力导向布局可视化：

- **渲染引擎**：Pixi.js + d3-force，在 Canvas 上高性能渲染大规模图数据
- **力导向布局**：d3-force 模拟物理力学，相关实体自动聚拢，不相关的实体相互排斥
- **社区着色**：不同社区用不同颜色标记，视觉上一目了然

支持缩放、拖拽、点击查看详情等交互操作。

![知识库问答总览](../../../images/readme/demo-008-knowledge-base-qa-overview.png)

## 实体搜索

图谱工作台顶部提供实体搜索框，输入实体名称后：

- 实时模糊匹配，显示候选实体列表
- 点击候选实体，画布自动平移到该实体并高亮显示
- 搜索结果同时展示实体的基本属性和所属社区

## 实体详情

点击画布上的实体节点，右侧详情面板展示：

- **属性列表**：实体的所有键值对属性
- **相邻关系**：与该实体相连的所有关系，包括关系类型和关联的实体名称
- **社区信息**：实体所属的社区 ID 和社区规模

## 社区分析

Leiden 算法在图谱构建时自动运行，检测实体聚类结构。在工作台中：

- 不同社区用不同颜色标记实体节点
- 社区面板列出所有检测到的社区，按规模排序
- 点击社区可高亮该社区的所有实体
- 每个社区生成一份文本摘要，描述社区内实体的共同特征

社区检测帮助理解图谱的整体结构，发现隐含的主题分组。

## 关系查询

图谱工作台支持路径查询：选择两个实体，查询它们之间的最短路径或所有路径。结果以高亮路径的形式在画布上展示，同时列出路径上经过的所有实体和关系。

## 构建图谱

### 从文件导入

上传文件（PDF、Markdown、纯文本）或直接粘贴文本内容，系统使用 GraphRAG 技术自动：

1. 提取文本中的实体（人名、组织、概念、术语等）
2. 识别实体之间的关系
3. 构建实体和关系写入图谱

提取过程在本地完成，不需要调用外部服务。

### Agent 构建

Agent 可以通过系统工具直接操作图谱：

| 操作 | 说明 |
|------|------|
| 列出图谱 | 获取所有图谱列表 |
| 创建图谱 | 新建 `.graph.db` 文件 |
| 删除图谱 | 删除图谱及所有数据 |
| 搜索实体 | 按名称模糊搜索实体 |
| 查看实体详情 | 获取实体的属性、关系和社区信息 |
| 创建实体 | 添加新实体，指定属性 |
| 更新实体 | 修改实体属性 |
| 删除实体 | 删除实体及其关联关系 |
| 创建关系 | 在两个实体间建立关系 |
| 查询关系 | 查询实体间的路径和关系 |
| 读取社区报告 | 获取社区检测结果和摘要 |
| 导入文件 | 从工作区文件提取实体和关系 |

### 多图谱支持

系统支持创建任意数量的图谱，每张图谱独立管理：

- 不同领域的知识分开存储（论文引用图谱、项目依赖图谱、知识分类图谱等）
- 图谱之间互不干扰
- 在资源面板中统一管理所有图谱

## 图谱问答

在工作台输入框中直接向图谱提问，系统会：

1. 在图中搜索与问题相关的实体
2. 提取这些实体的相邻关系和子图上下文
3. 返回相关实体列表和子图的可视化结果

这个功能适合快速探索图谱中的某个主题区域。

## 使用场景

- **论文引用分析**：导入论文元数据，构建引用关系图谱，发现研究脉络
- **项目依赖管理**：构建项目、模块、依赖库之间的关系图，理清架构
- **知识分类**：将领域知识中的概念和术语组织成图谱，辅助学习和检索
- **人物关系梳理**：从文档中提取人物和组织，构建关系网络
