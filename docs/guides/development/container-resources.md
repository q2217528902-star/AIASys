# Docker 沙盒

Docker 沙盒提供隔离的容器化执行环境，适合需要系统依赖或强隔离的场景。

![监控任务面板](../../../images/readme/panel-monitor.png)

## 什么时候用 Docker

默认的 Python 执行、Notebook 和脚本运行都走本地 UV 环境，不需要 Docker。

以下场景适合用 Docker：

- 代码依赖特定的系统库或工具链（如编译 C 扩展、调用外部二进制）
- 需要固定的 Linux 环境，与宿主系统隔离
- 需要限制代码对文件系统和网络的访问范围

如果只是跑 Python 代码、装 pip 包，用 UV 环境就够了，不需要切到 Docker。

## 两种模式

### 注册已有容器

提供已运行的 Docker 容器 ID 或容器名称，系统将其注册为可用的执行资源。

适合已有固定容器、不想每次重新创建的场景。

### 创建新容器

系统根据用户配置创建新的 Docker 容器。需要提供：

- **镜像名称**：Docker 镜像，如 `python:3.12-slim`
- **挂载路径**：容器内的工作区挂载点，默认 `/workspace`，映射到工作区文件系统
- **启动命令**：容器启动后执行的命令（可选）
- **环境变量**：注入容器的环境变量（可选）
- **自动启动**：是否在系统启动时自动拉起容器

## 容器操作

已注册的容器支持以下操作：

- **启动**：拉起已停止的容器
- **停止**：停止运行中的容器，不删除
- **查看日志**：查看容器标准输出和标准错误
- **注销/删除**：从系统中移除容器注册信息，可选择同时删除容器本身

## 容器状态

| 状态 | 含义 |
|------|------|
| 运行中 | 容器正常运行 |
| 已停止 | 容器已停止，可以重新启动 |
| 异常 | 容器运行出错，无法正常使用 |

## 挂载路径

挂载路径是容器内访问工作区文件的入口。默认 `/workspace`，创建容器时系统自动将当前工作区的文件系统路径挂载到此目录。

容器内的代码通过 `/workspace` 读写文件，与宿主的工作区文件实时同步。

## 访问入口

左侧 Activity Bar 的工作区设置面板，切换到"执行资源"标签页，再切换到"Docker 沙盒"子标签页。

![AIASys 工作区三栏布局](../../../images/readme/workspace-layout.png)

## 注意事项

- Docker 必须先在宿主机上安装并运行，系统不会自动安装 Docker
- 默认执行模式是本地 UV 环境，不需要主动切换到 Docker
- 容器内安装的包和修改的文件系统在容器删除后会丢失，持久化数据应放在挂载路径下
