# Notebook 执行

AIASys 内置 Jupyter Notebook 支持，可以直接在工作区中打开、编辑和运行 `.ipynb` 文件。

![Notebook 代码执行与图表预览](../../../images/readme/demo-003-notebook-analysis-overview.png)

## 打开 Notebook

在文件树中找到 `.ipynb` 文件，双击或右键选择"打开"，主画布会加载 Notebook 编辑器。

Notebook 编辑器以单元格为基本单位，每个单元格可以是 Markdown 单元格或代码单元格。页面顶部有工具栏，包含运行、保存、导出等操作按钮。

## 运行代码

### 运行单个单元格

点击代码单元格左侧的运行按钮（三角形图标），或选中单元格后按 `Shift+Enter`。

执行过程：

1. 单元格左侧出现运行指示器（旋转图标）
2. 输出区域显示执行结果
3. 运行完成后指示器变为完成状态（对勾或叉号）

### 运行全部单元格

点击工具栏的"全部运行"按钮，按顺序依次执行所有代码单元格。任一单元格执行失败时，后续单元格默认停止执行。

### 中断执行

运行中的单元格可以通过工具栏的"中断"按钮停止。中断后会保留已产生的输出。

## 输出支持

代码单元格的输出支持以下格式：

| 输出类型 | 显示方式 |
|----------|----------|
| 文本 | 纯文本输出，支持 ANSI 颜色 |
| HTML | 内嵌渲染 HTML 内容 |
| JSON | 格式化 JSON 显示，支持折叠/展开 |
| 图片（PNG/SVG） | 直接渲染显示，支持缩放 |
| 流式输出 | 实时追加文本，适合进度条等场景 |
| 错误回溯 | Python 异常完整调用栈，关键行高亮 |

输出区域可折叠，点击输出区域顶部的折叠按钮收起长输出。

## Markdown 单元格

Markdown 单元格支持：

- 标准 Markdown 语法（标题、列表、链接、表格、代码块等）
- LaTeX 数学公式（行内 `$...$` 和块级 `$$...$$`）
- 双击单元格进入编辑模式，显示原始 Markdown 源码
- 按 `Shift+Enter` 渲染并移动到下一个单元格

## Inspector 面板

Notebook 编辑器右侧提供四个 Inspector 面板，点击标签页切换：

### Variables（变量查看器）

显示当前内核中所有变量的名称、类型、值和大小。支持：

- 按名称搜索变量
- 展开查看复杂对象（列表、字典、DataFrame）的内部结构
- 变量更新时自动刷新

### Execution History（执行历史）

按时间倒序列出所有执行过的代码和对应的输出。每条记录包含：

- 执行时间
- 代码内容（截断显示）
- 执行状态（成功/失败）
- 点击记录可展开查看完整输出

### Diff（代码对比）

对比当前 Notebook 与上次保存版本之间的差异：

- 新增的单元格以绿色标记
- 删除的单元格以红色标记
- 修改的单元格显示逐行差异

### Outline（大纲）

以树形结构显示 Notebook 的结构：

- 按标题层级组织（Markdown 单元格中的 `#`、`##` 等）
- 点击标题跳转到对应单元格
- 代码单元格显示为无标题节点，标注行数

## Python 环境

### 环境绑定

Notebook 在工作区绑定的 Python 环境中运行。创建或切换工作区时指定的 Python 环境会自动成为 Notebook 的执行内核。

### 环境切换

在 Notebook 工具栏的内核选择器中可以切换到其他已注册的内核环境。切换后：

- 之前运行产生的变量和状态会丢失
- 需要重新运行单元格
- Variables 面板刷新为新内核的状态

支持的切换来源：

- 工作区 UV Python 环境（工作区默认）
- 系统已注册的 Jupyter 内核
- 其他工作区的 Python 环境（需显式注册）

### 依赖安装

如果代码中 `import` 的包在当前环境中不存在，Notebook 不会自动安装。需要在工作区设置中通过 UV 安装对应包，或在代码单元格中手动执行 `!uv pip install <package>`。

## Agent 交互

Agent 可以直接操作 Notebook：

- 读取单元格内容：Agent 可以查看所有单元格的源码和输出
- 编辑单元格：Agent 可以修改 Markdown 或代码单元格的内容
- 运行代码：Agent 可以触发单元格执行并读取输出结果
- 添加/删除单元格：Agent 可以在指定位置插入或删除单元格

在对话侧栏中，当 Agent 引用了 Notebook 中的代码或输出时，点击引用链接会跳转到对应的单元格。Agent 执行 Notebook 操作的完整记录会保留在会话历史中。
