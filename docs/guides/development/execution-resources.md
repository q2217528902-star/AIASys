# 执行资源

执行资源管理当前工作区的 Python 运行环境，支撑 Notebook 和代码执行。

## 概念

工作区可以绑定一个或多个 Python 环境。这些环境是 Notebook 和代码执行的基础设施，Notebook 的内核选择器会列出所有已绑定的环境，随时可以切换。

## 环境类型

### UV 环境

系统通过 `uv` 自动创建和管理的独立虚拟环境。创建时指定 Python 版本和需要的包，系统自动完成安装和隔离。

适合不想手动管理环境的场景。UV 环境是纯 Python 环境，不需要 Docker，也不需要用户自己准备 Python 解释器。

### 已注册 Python

用户提供已有 Python 解释器的路径（如 conda 环境、系统 Python），系统将其注册到执行资源中。

适合已有固定 Python 环境、想复用的场景。系统只记录路径，不会修改原始环境。

## 环境状态

| 状态 | 含义 |
|------|------|
| 已注册 | 刚注册，尚未验证可用性 |
| 就绪 | 已验证，可以使用 |
| 运行中 | 当前有 Notebook 或代码正在该环境中执行 |
| 已停止 | 执行结束，空闲中 |
| 缺失 | 注册的 Python 路径不存在或无法访问 |
| 不可用 | 环境存在但无法正常执行代码 |
| 异常 | 执行过程中发生错误 |

## 概览页

打开"执行资源"面板后，顶部显示当前执行入口信息（本地 Python 或 Docker），下方有 4 个指标卡片：

- Python 环境状态：当前就绪/总数
- Docker 活跃数：运行中的容器数量
- 工作区变量数：当前工作区绑定的环境变量数量
- 已注册环境数：所有已注册的 Python 环境总数

## 注册 UV 环境

1. 点击"启用 Python"或"新建环境"
2. 选择 Python 版本（如 3.12）
3. 选择需要的包（可选，创建后也可追加安装）
4. 系统自动检测 `uv` 是否可用，未安装时自动安装
5. 创建完成后环境进入"就绪"状态

## 注册已有 Python

1. 提供 Python 可执行文件的完整路径
2. 填写 `env_id`（环境标识）和 `display_name`（显示名称）
3. 系统验证路径是否有效
4. 验证通过后环境进入"就绪"状态

## 环境绑定

环境注册后需要绑定到当前工作区。绑定后，Notebook 的内核选择器会列出该环境。

一个工作区可以绑定多个环境，一个环境也可以绑定到多个工作区。

## 解除绑定与注销

- **解除绑定**：断开环境与工作区的关联，不影响环境本身，其他工作区仍可使用
- **注销**：从系统中彻底删除环境注册信息，所有工作区的绑定关系同时清除

## 包安装

UV 环境支持在面板中直接安装 Python 包。

操作方式：
1. 在执行资源面板找到目标 UV 环境
2. 点击环境卡片展开详情（显示已安装包列表，最多展示 24 个）
3. 在输入框中输入包名，支持多个包用空格分隔
4. 点击安装，系统通过 `uv pip install` 安装到对应环境中

已注册 Python 环境不支持通过面板安装包，需要用户在原始环境中自行管理。

## 内核环境管理

系统可以列出所有已注册的 Jupyter 内核规范（kernel specs），包括：

- 内核名称
- 对应的 Python 路径
- 内核状态

支持的操作：

- 注册新内核：提供内核名称和 Python 路径
- 删除内核：移除不再需要的内核
- `python3` 内核受保护，不可删除

## 访问入口

左侧 Activity Bar 的工作区设置面板，切换到"执行资源"标签页。

## 与 Notebook 的关系

Notebook 的内核选择器列出所有已绑定到当前工作区的环境。用户可以在 Notebook 运行时随时切换内核，切换后之前运行产生的变量和状态会丢失，需要重新执行单元格。
