# 终端与监控任务

终端和监控任务是 AIASys 左侧活动栏中的两个独立功能模块，分别负责交互式命令执行和后台任务管理。

## WebSocket 终端

左侧活动栏点击"终端"图标打开 WebSocket 终端面板。终端直接连接到当前工作区的文件系统，不是独立容器。

终端支持以下操作：

- 执行任意 shell 命令
- 运行脚本（Python、Shell 等）
- 管理 Python 虚拟环境和依赖
- 查看和编辑工作区文件

终端会话绑定到当前工作区的工作目录。切换工作区时，终端会自动 cd 到新工作区的根目录。终端会话在页面刷新后保持连接，不需要重新打开。

### 跨平台支持

| 平台 | 实现方式 | 说明 |
|------|----------|------|
| Linux | POSIX openpty + fork/exec | 标准伪终端 |
| macOS | POSIX openpty + fork/exec | 标准伪终端 |
| Windows | pywinpty (ConPTY/WinPTY) | 依赖 `pywinpty>=2.0.0`，安装桌面版时自动包含 |

Windows 终端优先查找 PowerShell，回退到 cmd.exe。pywinpty 未安装时终端功能不可用，系统会提示安装命令。

## 监控后台任务

Agent 执行长时间任务时，可以把任务放到后台运行，不阻塞对话流。典型场景：

- 模型训练
- 大规模数据处理
- 定时数据采集
- 长时间运行的脚本

### 后台任务操作

**spawn**：Agent 通过工具调用启动后台进程。进程的 stdout 和 stderr 会自动重定向到文件，存储在 `{workspace}/.aiasys/tasks/` 目录下。

**poll**：Agent 可以随时检查后台任务的输出文件和进程状态，获取最新的运行日志。

**kill**：终止指定的后台任务。Agent 或用户都可以手动终止不再需要的后台进程。

### 监控面板

左侧活动栏点击"监控任务"图标打开监控面板。面板显示所有后台任务的列表，每项包含：

- 任务名称和启动时间
- 运行状态（运行中 / 已完成 / 已终止 / 异常退出）
- 展开后可查看实时输出日志
- 手动终止按钮

## 设计哲学

重型任务异步化，不阻塞对话流。终端和监控任务的设计前提是：用户不应该因为一个模型训练跑了两小时就没法继续跟 Agent 对话。长任务放到后台，Agent 定期检查进度，用户随时可以切换到其他工作。
