# 多维表格

多维表格是类似 Notion Database 的结构化数据管理工具。每张表格是一个独立的 SQLite 数据库文件（`.table.db`），存储在工作区目录下。

![多维表格实验记录](../../../images/readme/demo-data-table.png)

## 基本概念

多维表格的核心设计：

- 每张表格有多个**列**，每列有固定的数据类型
- 每行是一条**记录**，记录包含所有列的值
- 表格文件是完整的 SQLite 数据库，可以被 Agent 直接读写
- 表格与文件树中的其他文件地位相同，支持预览、移动、删除

## 创建表格

在文件树工具栏点击"新建"，选择"多维表格"，弹出创建对话框：

1. 输入表格名称（自动添加 `.table.db` 后缀）
2. 定义列结构，每一列需要指定：
   - **列名**：字段名称
   - **类型**：数据类型，创建后可按需修改
   - **必填**：是否要求每条记录必须有值

点击"创建"后，表格文件在工作区目录下生成，并在主画布区域自动打开。

## 列类型

支持以下列类型：

| 类型 | 说明 | 示例 |
|------|------|------|
| 文本 | 单行文本 | 任务标题、备注 |
| 数字 | 整数或小数 | 金额、数量 |
| 日期 | 日期选择器 | 截止日期、创建时间 |
| 单选 | 从预设选项中选择一个 | 状态（待办/进行中/已完成） |
| 多选 | 从预设选项中选择多个 | 标签 |
| 复选框 | 布尔值，勾选或未勾选 | 是否完成、是否紧急 |
| 文件 | 关联工作区中的文件 | 附件、截图 |
| 链接 | URL 链接 | 参考资料、外部文档 |

单选和多选类型在定义列时需要预先设置选项列表，之后可在列管理中修改。

## 列管理

打开表格后，点击表头右侧的下拉箭头进入列管理：

- **添加列**：在表格右侧点击"+"按钮添加新列
- **删除列**：在列管理中选择"删除列"，确认后该列及所有数据被移除
- **修改列类型**：在列管理中更改类型。类型变更会尝试转换现有数据，不兼容的数据可能丢失

列管理也展示列的元信息：名称、类型、是否必填、选项列表（单选/多选）。

## 数据网格

表格主区域以网格形式展示记录：

- 每行一条记录，每列一个单元格
- 默认分页显示，每页 50 条，可翻页浏览
- 点击单元格进入编辑模式，直接修改内容
- 按 `Tab` 跳到下一列，按 `Enter` 确认编辑并跳到下一行

### 添加行

点击表格底部的"+"按钮添加新空白记录。新行自动分配 ID，所有列初始为空。

### 删除行

每行末尾有删除按钮（垃圾桶图标）。点击后弹出确认对话框，确认后该记录被永久删除。

## Schema 视图

表格提供 Schema 视图，以结构化方式展示所有列的元数据：

- 列名
- 数据类型
- 是否必填
- 单选/多选的选项列表

Schema 视图适合在设计阶段快速浏览表格结构，或与 Agent 协作时确认字段定义。

## Agent 交互

Agent 可以通过系统内置工具直接操作多维表格：

- **创建表格**：Agent 可以创建新的 `.table.db` 文件，指定列定义
- **读取 Schema**：Agent 可以读取表格的列结构
- **读写记录**：Agent 可以查询、新增、修改、删除记录
- **管理列**：Agent 可以添加和删除列

Agent 操作表格时，表格文件被当作普通数据源处理，不依赖额外的 API 层。

## 文件格式

多维表格存储为 `.table.db` 文件，本质是 SQLite 数据库。可以直接用 SQLite 命令行工具或编程语言打开查看：

```bash
sqlite3 my_table.table.db ".schema"
sqlite3 my_table.table.db "SELECT * FROM records LIMIT 10"
```

表格文件在工作区文件树中显示，支持常规的文件操作（复制、移动、重命名、删除）。

## 使用场景

- **对比矩阵**：多维度对比不同方案、工具或产品的特性
- **实验记录**：记录实验参数、结果和备注，支持按列筛选
- **数据整理**：结构化存储调研数据，方便后续查询和分析
- **需求跟踪**：用单选列标记状态，多选列打标签，跟踪需求推进进度
