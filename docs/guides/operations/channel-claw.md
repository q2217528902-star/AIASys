# IM 通道 / Claw 连接器

Claw 是工作区和会话的外部通信通道。它让 AIASys 可以通过微信等 IM 平台接收任务和发送通知。

![频道接入面板](../../../images/readme/channel%20management.png)

## Claw 的定位

Claw 不是独立工作区。它复用当前工作区的 Soul、项目档案、工作区记忆和会话上下文，只是在原有的 Web/桌面交互方式之外增加了一条 IM 通道。Agent 工具目录中不提供 Claw 专属工具，这是设计决策。

Claw 额外提供的能力：

- 微信 / IM 连接
- 消息收发
- 附件导入
- 远程确认

## 通道配置

通道配置使用 YAML 文件驱动，在通道管理界面中编辑。每个 IM 平台对应一个通道配置，包含平台类型、连接参数和消息路由规则。

## 扫码登录

微信通道通过扫描二维码登录。在通道管理界面中点击微信通道的"登录"按钮，系统生成二维码，用微信扫描后完成授权。

## 远程任务分发

通过 IM 工具向 AIASys 发送任务指令，系统接收后创建对应的会话或 AutoTask 并开始执行。执行完成后通过同一通道发送通知。

任务分发的消息格式和路由规则在通道配置中定义。

## 消息收发

Agent 可以通过绑定的通道发送和接收消息。发送消息时指定目标通道和接收方，接收消息时按通道配置的路由规则分发到对应会话。

## 设计边界

Claw 与外部 IM 频道之间不做强一致性同步。外部频道只是信息发送渠道，两边的通信上下文、节奏和参与者完全不同，不需要也不应该保证内容完全一致。前端不保留手动补发或推送入口。

外部消息进入 AIASys 后，按当前工作区的上下文独立处理。IM 中的对话历史和 AIASys 中的会话历史是两条独立的线。
