# 会话管理（高级功能）

会话是工作区内的一条任务推进线。每个会话有独立的对话历史、执行日志和上下文。本文档覆盖会话的高级管理功能。

![AIASys 工作区三栏布局](../../../images/readme/workspace-layout.png)

## 会话 Fork

从对话历史的任意节点创建新会话。新会话继承分叉点之前的完整历史，之后两条会话各自独立推进。

使用场景：

- 想探索不同方向，又不想污染主会话
- 某个方向的尝试不理想，回到主会话继续

Fork 后的会话是独立对象，互不影响。原会话和新会话各自有自己的 token 预算和 checkpoint。

## 会话导出（Bundle）

将完整会话历史打包导出为可下载的 bundle 文件。导出内容包含：

- 完整对话记录
- 执行日志
- 会话期间生成的文件快照

Bundle 文件可以在其他 AIASys 实例中导入恢复。

## 会话预算

会话的 token 预算记录在会话元数据中，限制该会话的累计 token 消耗。预算适用于普通对话和 AutoTask，消耗达到上限后 Agent 执行被阻塞。

预算设置入口在对话栏顶部的 TokenUsageBar 中。修改预算不影响已消耗的 token 统计，只改变上限值。

![上下文预算窗口](../../../images/readme/context%20window.png)

## Checkpoint

系统在执行过程中自动创建 checkpoint 标记，用于内部状态管理和上下文恢复。这些标记出现在消息历史中，帮助 Agent 追踪关键节点。

当前阶段 checkpoint 是系统内部机制，不提供用户手动创建、浏览或恢复的界面入口。

## 会话生命周期

```
创建 → 活跃 → 暂停/停止 → 归档
```

归档后的会话仍然可读，但不再消耗预算，Agent 也不能继续在该会话中执行任务。

## Token 用量显示

对话栏顶部的 TokenUsageBar 实时显示三项信息：

- 当前上下文占用（本轮提示词大约占用的 token 数）
- 模型上下文窗口大小
- 会话预算使用比例

上下文占用和会话预算不是同一个概念。上下文占用只表示当前轮占用了多少窗口，预算跟踪的是整个会话的累计消耗。
