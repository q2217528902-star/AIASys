# Agent 配置

> 当前版本: v0.4.27

本文档说明 Agent 的三层配置体系：Soul（用户默认）、项目画像（工作区）、会话覆盖（会话）。

![模型配置弹窗](../../../images/readme/demo-model-config-panel.png)

## 三层配置概述

Agent 的行为由三个配置层叠加决定，优先级从高到低：

| 层级 | 作用域 | 配置内容 |
|------|--------|----------|
| Agent Soul | 用户默认（跨工作区） | 协作风格、表达方式、长期偏好 |
| 项目画像 | 当前工作区 | 项目边界、领域知识、任务上下文 |
| 会话覆盖 | 当前会话 | 执行策略、临时调整 |

优先级规则：**会话覆盖 > 项目画像 > Agent Soul**。高层配置覆盖低层同名字段，没有配置的字段继承下层。

### Agent Soul

Soul 定义了 Agent 在所有工作区中的基础行为模式，是跨工作区的稳定配置。适合配置的内容：

- 回复风格偏好：简洁还是详细，偏技术还是偏通俗
- 语言偏好：默认使用什么语言回复
- 执行习惯：是否默认在执行前做规划，是否在执行完成后主动汇报
- 代码风格：偏好什么编程语言、命名规范等

Soul 配置存储在用户默认全局工作区中，所有工作区共享。

### 项目画像

项目画像是工作区级别的配置，描述当前项目的边界和 Agent 需要了解的领域信息。适合配置的内容：

- 项目简介：这个工作区在做什么
- 技术栈约束：项目使用的框架、语言版本等
- 领域知识：Agent 需要了解的业务背景、术语定义
- 边界规则：明确告诉 Agent 不要做什么（如不要修改某些文件、不要调用某些工具）

项目画像配置存储在工作区的 `.aiasys/agent_config/` 目录下。

### 会话覆盖

会话覆盖是对当前会话的临时调整，不影响其他会话和后续新会话。适合配置的内容：

- 本次会话的特殊目标
- 临时改变执行风格（如本次要求更激进或更保守）
- 实验性的配置调整

会话覆盖配置存储在会话目录中，会话结束后不再生效。

## 配置入口

点击左侧 Activity Bar 底部的工作区设置图标（齿轮），在设置面板中找到"Agent 配置"分类。面板中分为三个区域：

- **Agent Soul**：编辑用户默认配置
- **项目画像**：编辑当前工作区配置
- **会话覆盖**：编辑当前会话配置

每个区域提供文本输入框，支持 Markdown 格式。修改后即时保存。

![能力管理面板](../../../images/readme/panel-capability-management.png)

## Agent 如何读取配置

Agent 在每轮执行前读取三层配置，按优先级合并后确定本轮的行为准则。具体流程：

1. 读取 Agent Soul，作为基础行为约束
2. 读取项目画像，覆盖和补充工作区特定规则
3. 读取会话覆盖，应用当前会话的临时调整
4. 将合并后的配置注入 Agent 的提示词上下文

三层配置的总内容量有上限，超过上限时会截断低优先级层的内容。建议每层控制在合理范围内，把最核心的约束放在最高优先级层。

## 配置文件位置

### Agent Soul

```
workspaces/{user_id}/global_workspace/.aiasys/agent_config/soul.md
```

### 项目画像

```
workspaces/{user_id}/{workspace_name}/.aiasys/agent_config/project_profile.md
```

### 会话覆盖

```
workspaces/{user_id}/{workspace_name}/.aiasys/sessions/{session_id}/agent_override.md
```

配置文件为 Markdown 格式，Agent 直接读取原始内容作为提示词约束。不建议在配置中使用复杂的 Markdown 嵌套结构，Agent 解析时可能丢失层次关系。推荐用简洁的段落和列表组织内容。

## 配置示例

### Agent Soul 示例

```markdown
- 用简体中文回复
- 代码注释用中文
- 执行前先做简要规划，完成后汇报结果
- 优先使用 Python 编写脚本
- 回复风格简洁，不添加无关信息
```

### 项目画像示例

```markdown
本项目是一个用户行为分析系统，使用 FastAPI + React + PostgreSQL 技术栈。

关键术语：
- "事件"指用户在产品中的操作记录
- "漏斗"指从注册到付费的转化路径

不要修改 apps/backend/alembic/ 目录下的迁移文件。
数据库操作必须通过 ORM，不要直接写 SQL。
```

### 会话覆盖示例

```markdown
本次会话聚焦于性能优化，优先关注响应时间而非功能完整性。
遇到不确定的方案直接选择并执行，不需要 AskUser 确认。
```
