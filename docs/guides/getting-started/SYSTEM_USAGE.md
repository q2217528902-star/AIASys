## 1. 系统概述

AIASys 是一个 AI Agent 工作平台，提供本地单机单用户的智能分析工作台。核心能力：

- **多 Agent 协作**：任务自动分解，多个 Sub Agent 并行执行
- **代码执行**：内置 IPython 运行环境，支持数据分析与可视化
- **知识管理**：内置 SQLite + sqlite-vec 知识库 + GraphRAG 知识图谱
- **数据查询**：支持 DuckDB 及外部数据库连接
- **MCP/Skill 市场**：插件式能力扩展

当前版本按**本地单用户**设计，不包含团队协作、多用户权限等企业功能。

## 2. 启动系统

请先完成 [快速启动指南](QUICKSTART.md) 中的依赖准备步骤。

### 启动开发环境

```bash
cd AIASys
./dev.sh
```

服务启动后：

| 地址 | 说明 |
|------|------|
| `http://localhost:13000` | 前端首页 |
| `http://localhost:13000/workspace` | 分析工作区（主界面） |
| `http://localhost:13001/health` | 后端健康检查 |

### 手动启动

```bash
# 后端（注意：不要加 --reload，避免多 agent 时热重启干扰）
cd apps/backend
.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 13001

# 前端
cd apps/web
npm run dev -- --host 0.0.0.0 --port 13000
```

## 3. 界面导览

### 3.1 首页

访问 `http://localhost:13000` 进入首页，点击"开始分析"进入工作区。

### 3.2 三栏布局

进入工作区后采用三栏布局：

```
左侧导航栏 | 中间主画布 | 右侧对话侧栏
```

- **左侧导航栏**：工作区切换、搜索、文件树、资源面板入口
- **中间主画布**：文件管理、资源库、数据查询、知识图谱、Notebook 等
- **右侧对话侧栏**：当前会话对话、会话列表、输入区

### 3.3 顶部导航

分析页顶部仅显示当前工作区和会话标题。全局控制面板（LLM 配置、Agent 配置、AutoTask、Monitor 等）通过首页导航或工作区侧栏底部设置菜单进入。

## 4. 工作区管理

### 4.1 创建工作区

1. 在 `/workspace` 页面点击"新建工作区"
2. 输入工作区名称
3. 选择是否启用 Python 运行环境；不启用时不会创建工作区 `.venv`

### 4.2 切换工作区

左侧导航栏顶部显示当前工作区，点击可展开工作区列表进行切换。

### 4.3 工作区设置

在工作区中可配置：
- LLM 模型选择（全局 / 工作区 / 会话三级覆盖）
- Python/UV 运行环境
- Docker 沙盒资源（需要系统依赖、固定 Linux 工具链或强隔离时显式登记或创建）
- 资源挂载配置

---

## 5. 文件与资源

### 5.1 文件管理

中间主画布的"工作文件"标签页提供：

- **文件浏览**：树形视图展示工作区文件
- **文件操作**：新建文件/文件夹、重命名、删除、拖拽移动
- **文件上传**：拖拽上传本地文件到工作区
- **文件编辑**：支持 `.md`、`.txt`、`.json`、`.yaml`、`.ipynb` 等文本文件在线编辑
- **文件导出**：支持导出为 Markdown 文档

### 5.2 资源库

"资源库"标签页展示工作区全局资源：

- **知识库**：SQLite + sqlite-vec 向量知识库，支持文档导入与语义检索
- **数据库**：内置 DuckDB 及外部数据库连接
- **图谱**：GraphRAG 知识图谱，支持实体、社区查询

## 6. Agent 对话与执行

### 6.1 发起对话

在右侧对话侧栏输入区输入问题，Agent 将在工作区上下文中执行任务。

### 6.2 会话管理

每个工作区可创建多个执行会话。会话之间上下文隔离，可并行推进不同任务线。

- **新建会话**：点击会话列表顶部"+"按钮
- **切换会话**：点击会话名称
- **会话名自动生成**：AI 根据对话内容自动摘要

### 6.3 模型选择

点击输入区上方的模型选择器，可在以下级别配置模型：

- **全局**：对所有工作区生效
- **工作区**：仅对当前工作区生效（覆盖全局）
- **会话**：仅对当前会话会话生效（覆盖工作区）

### 6.4 附件上传

对话输入区支持上传附件文件，Agent 将使用这些文件作为本轮对话的上下文。

### 6.5 执行流

右侧侧栏实时展示 Agent 的工具调用和执行进展，包括：

- **Sub Agent 分配**：任务分解后的子 Agent 执行情况
- **工具调用**：文件读写、代码执行、数据库查询等
- **执行树**：多 Agent 并行时的执行层级关系

## 7. Notebook 执行

### 7.1 打开 Notebook

在文件树中点击 `.ipynb` 文件，中间主画布进入 Notebook 编辑器。

### 7.2 运行代码

- 点击单元格左侧运行按钮执行代码
- 支持 Python 代码
- 输出支持：文本、HTML、JSON、图片（PNG/SVG）、流式输出
- 代码执行环境与工作区绑定

### 7.3 Inspector 面板

Notebook 提供四个检查面板：

- **变量**：查看当前内核中的变量及其值
- **运行记录**：查看历史执行记录
- **差异对比**：对比代码变更
- **大纲**：Notebook 结构概览

## 8. 知识库与图谱

### 8.1 知识库管理

在 `/workspace` 的资源入口或知识库弹层中进入：

- **文档导入**：上传 PDF、Markdown 等文档
- **语义检索**：输入问题，返回相关知识片段
- **知识库列表**：浏览已创建的知识库及其文档

### 8.2 知识图谱

在 `/workspace` 的资源入口或知识图谱弹层中进入：

- **图谱可视化**：力导向布局展示实体关系
- **实体搜索**：按名称检索知识实体
- **社区分析**：自动识别聚类社区
- **关系查询**：查询两个实体之间的路径

### 8.3 图谱构建

通过"构建"标签页：

- 上传文本文件，系统自动提取实体和关系
- 配置 LLM 用于图谱构建

## 9. 数据库查询

### 9.1 内置 DuckDB

系统内置 DuckDB 实例，所有工作区默认可用。在"数据查询"面板中：

1. 选择 DuckDB 连接
2. 浏览表结构和字段
3. 编写并执行 SQL 查询
4. 查询结果可传递给 Agent 作为上下文

### 9.2 外部数据库

在"数据库连接"中配置外部数据库：

- 支持的连接器类型取决于系统配置
- 连接信息按用户存储，可在会话中挂载使用
- 查询结果可导出为文件

## 10. AutoTask 自动任务

### 10.1 创建自动任务

在 `/workspace` 的工作区侧栏打开 `自动任务`：

1. 点击"新建任务"
2. 选择触发方式：单次、周期、Cron 或连续推进
3. 设置任务标题、描述和 Prompt
4. 选择绑定当前会话或每次新建会话

### 10.2 查看执行记录

- 任务列表中查看最近执行状态
- 绑定会话后可在当前会话历史和执行记录里查看运行结果

## 11. 协作专家启用

### 11.1 协作专家市场

在 `/workspace` 左侧边栏打开 `能力管理`，在其中的`专家协作节点`分类下管理协作专家市场、我的默认启用和当前工作区启用：

- **协作专家市场**：查看系统提供的专家候选
- **我的默认启用**：配置跨工作区继承的协作专家
- **当前工作区启用**：配置当前工作区实际可派发的协作专家

### 11.2 协作专家与 Agent

在对话中，主控 Agent 会根据任务类型派发已启用的协作专家。协作专家定义了：

- 系统提示词（Agent 的行为模式）
- 可用工具（该专家被允许调用的工具）

## 13. LLM 模型配置

### 13.1 全局配置

当前默认优先使用 Step 系列模型。首次启动前可以在 `apps/backend/config.toml` 中预置 `default_provider`、`default_model`、服务商 `Base URL` 和 API Key，并为每个模型声明 `capabilities` 以区分纯文本模型和多模态模型。用户配置为空时，后端会把这里的 `llm.providers` 同步到当前用户的全局工作区模型配置。

### 13.2 用户级配置

在 `/workspace` 页面打开左侧边栏底部的`工作区设置`，进入`模型配置`：

- 添加、编辑、删除 LLM 服务商
- 测试 API Key 和 Base URL 是否可用
- 获取或手动添加模型
- 设置默认 Chat 模型和默认 Embedding 模型

DashScope、OpenAI、Anthropic 等其他服务商可以按协议补充接入。默认示例和首次启动推荐优先配置 Step 系列模型。

### 13.3 工作区/会话级配置

在分析页面输入区上方的模型选择器中，可针对工作区或会话覆盖默认模型。

## 推荐体验路径

1. 打开 `http://localhost:13000` → 点击"开始分析"
2. 创建一个工作区（命名为"体验"）
3. 在输入区输入一个简单任务，如："帮我分析一个 CSV 文件"
4. 观察右侧执行流，了解 Agent 如何分解任务
5. 在文件树中查看 Agent 生成的文件
6. 打开资源库查看可用的数据库和知识库
7. 尝试上传一个文档到知识库，体验语义检索
8. 在模型选择器中切换不同 LLM 提供商
9. 在 `/workspace` 左侧边栏打开 `能力管理`，在`专家协作节点`分类下查看协作专家市场和当前工作区启用状态
10. 在 `/workspace` 打开模型与工具配置，了解 Agent 配置选项

## 常见问题

### Q: Agent 回复慢？
A: 检查 LLM 配置中的提供商和模型，可切换到更快的模型（如 Kimi 系列）。

### Q: Docker 沙盒不可用？
A: 先确认 Docker 已安装并运行，再到工作区的 Docker 沙盒资源中登记已有容器或按镜像创建容器。默认 Python、Notebook 和普通脚本走本地 UV 运行环境，不需要把 `sandbox.default_mode` 改成 Docker。

### Q: 如何清除工作区数据？
A: 在工作区设置中可删除工作区及其所有数据。

### Q: 如何备份？
A: 工作区数据存储在 `apps/backend/data/` 目录下，直接备份该目录即可。
